#### 责编的"5 视角"即四专家的吸收体现

重构后责编（加载 `craft/editing.md`）在 7 步 SOP 的 Step 3（多元视角复查）中，自动走以下 5 个视角：

| 视角 | 继承自 |
|---|---|
| 节奏党（张哥） | 原 4 读者之一（节奏/爽感/钩子） |
| 文学派（林小姐） | 原 4 读者之一（语言/意象/留白） |
| 编辑（老周） | 原 4 读者之一（结构/市场） |
| 剧作家（老陈） | 原 4 专家之一（结构/节奏/反相位） |
| 教练（K 教练） | 原 4 专家之一（角色语言/对话/破防） |

责编通过 `editing.md` 第三节"多元思考训练"，在一个 Agent 内模拟 5 种视角——用 Token 效率换取独立性（27K → 7K/集）。

#### 新架构下"专家顾问"的位置

在 8 人班子中，原"专家"的职责由以下角色承担：

| 新架构角色 | 加载的 craft 文件 | 对应专家职责 |
|---|---|---|
| **编剧** | conflict + scene-design + mystery | 结构 + 钩子规划（老陈 + Q 老师） |
| **悬疑顾问** | mystery | 三铁律 + 线索三明治（Q 老师） |
| **表演指导** | characterology + dialogue | 角色身体 + 对话（K 教练） |
| **文学顾问** | prose | 意象 + 留白 + 破防戏（许教授 + K 教练） |
| **责编** | editing + prose + dialogue | 综合审查（4 专家合一） |

**每个专家的"待执行建议"不再是 TODO 清单——而是被班子成员在工作流中自动执行。**

例如：
- 旧版 "老陈建议：EP08 起强制标注 beat-sheet 每场景的情绪起点/终点数值" → 新版编剧在 `conflict.md` 附录 B 的 beat-sheet v3 模板中**默认**标注 `emotion_level` 和 `previous_scene_level`
- 旧版 "Q 老师建议：hooks-ledger 每集更新" → 新版责编在 Phase 4 wrap 前**自动**更新 hooks-ledger.md
- 旧版 "许教授建议：imagery-ledger 每集更新" → 新版文学顾问在 Phase 4 wrap 前**自动**更新 imagery-ledger.md
- 旧版 "K 教练建议：SOUL.yaml 补 `unique_markers` 字段" → 新版 drama-world 的角色 SOP 模板已内置该字段

#### 未来如需新增方法论

若通过多集实战复盘发现新的重要方法论：

1. 不再追加到本文件
2. 直接写入 `drama-director/references/craft/` 下对应的专业文件
3. 在本文件的"历史沉淀日志"表中追加一行记录（含来源 + 时间 + 迁入位置）
4. 对应的 Skill（drama-director）在下次加载时自动获得新知识

**知识的事实源是 `craft/`，索引层是本文件和 CODEBUDDY.md。先改事实源，再改索引。**
