# drama-brainstorm · 头脑风暴编排 Skill

## 适用场景

- 基于某个灵感/题材/参考作品，设计一部全新的小说
- 需要打破单一视角的思维定势
- 需要从世界观 / 角色 / 商业三个维度同时拉齐

## 核心输出

```
docs/<story-slug>-design.md           # 整合三视角的设计文档
docs/<story-slug>-characters.md       # （可选）扩展角色体系
```

## 标准流程

### Phase 1: 收集灵感种子

向用户确认：
- **核心灵感**：一句话描述（"考古+悬疑+神秘力量"）
- **参考作品**：1-3 部对标（"盗墓笔记 + 诡秘之主风味"）
- **改造自由度**：保留骨架 / 大幅改造 / 完全自由
- **目标产出**：设计文档 / 项目结构 / 样章

### Phase 2: 三视角并行

**同时**发起三个 Task 调用（不要串行）：

#### 2.1 世界观架构师（architect agent）

提示词模板：
```
你是网络小说世界观架构师。基于灵感种子「{seed}」，提出 3 个大胆的世界观改造方案。

每个方案必含：
1. 核心设定一句话（电梯演讲）
2. 世界观层次结构（显性/隐性/更深层）
3. 力量体系（是否有等级？如何量化成长？）
4. 多方势力（至少 3 方对抗）
5. 主角金手指（独特优势 + 代价）
6. 独特卖点（与同类差异化）
7. 爽点来源
8. 风险评估（哪里可能写崩）

爆款世界观共同特征：
- 洋葱结构（层层揭秘）
- 可量化的等级/进度
- 有限金手指但有明确代价
- 多方势力的利益冲突
- 可持续升级空间

每方案 300-400 字，直接输出不要寒暄。
```

#### 2.2 角色设计师（planner agent）

提示词模板：
```
你是网络小说角色设计专家。基于灵感种子「{seed}」，设计让读者欲罢不能的角色群像。

输出四部分：
A. 主角方案 3 个候选（人设一句话/独特标签/内在驱动/金手指代价/口头禅/为什么上头）
B. 反派/对手方案 3 个（必须人格化，身份/关系/魅力点/对抗价值）
C. 核心女性角色 2 个（不能花瓶，身份/动机线/张力/磕点）
D. 让人意外的关系 2-3 个（卧底盟友/身份反转/等）

每方案 150-250 字。
```

#### 2.3 市场编辑（planner agent）

提示词模板：
```
你是资深网文编辑/市场策划。基于灵感种子「{seed}」，给出商业方案。

输出六部分：
1. 类型定位 3 种（X+Y组合，核心卖点，目标平台，对标作品）
2. 开篇设计（前 3 章钩子，第一章前 500 字四要素）
3. 爽点体系 5 个可复用模板（类型/触发/结构/示例）
4. 节奏钩子 3 种章末钩子模板
5. 差异化卖点（vs 同类作品一句话）
6. 目标读者画像 3 类（人群/付费意愿/偏好）

直接输出务实建议，不要寒暄。
```

### Phase 3: 整合产出

收到三位专家的反馈后，主 Agent 执行：

1. **交叉比对**：找到三方共同指向的"最佳方向"
2. **合并设计**：
   - 世界观 = 架构师的某个方案（+ 市场的商业化改造）
   - 角色群像 = 设计师的最佳组合（+ 架构师的力量体系锚定）
   - 节奏爽点 = 市场编辑的模板 + 角色的冲突结构
3. **输出一份整合文档**：`docs/<story-slug>-design.md`

### Phase 4: 角色扩展（可选）

如果用户反馈"角色太少"：
- 调用批次角色设计
- 按 S(5-8) / A(10-15) / B(20-30) / C(30-50) 分级扩展
- 总数控制在 65-100 人
- 输出到 `docs/<story-slug>-characters.md`

### Phase 5: 开工衔接

与用户确认后可执行：
```bash
# 从设计文档初始化项目骨架
node scripts/init-from-design.js --name <slug> --from docs/<slug>-design.md

# 批量导入角色（需先准备 character-pack.yaml）
node scripts/import-characters.js --story <slug> --from <pack>.yaml
```

## 质量要求

- ⚠️  三个 Task 必须**并行**发起，不要等第一个返回再发第二个
- ⚠️  每个专家的输出要量化、可执行，不能是"建议考虑……"式模糊表达
- ⚠️  整合文档必须给出**明确的选择**（不是方案列表），方便用户一句话拍板
- ⚠️  世界观 / 角色 / 商业三者必须内部自洽（爽点服务于人物弧光，人物弧光服务于世界观）

## 反模式

- ❌ 串行调用三个专家（浪费时间）
- ❌ 把三份报告直接拼接作为产出（没有整合）
- ❌ 只输出方案列表让用户选（应该主动拍板）
- ❌ 宽泛的建议（"可以加更多元素"）
