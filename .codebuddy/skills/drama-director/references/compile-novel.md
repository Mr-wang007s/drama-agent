# 小说编译规范

> 从 Agent 回合制交互记录改写为**第三人称戏剧白描**叙事文本。Phase 3 的默认编译格式。

---

## 🔴 硬约束 · 第三人称叙事视角

### 核心定义

novel.md 的**叙述者视角**必须是**第三人称**——"他/她"做 X，"他/她"说 Y，"他/她"看见 Z。

### 明确禁止

| 禁止 | 示例 | 替代 |
|---|---|---|
| ❌ 叙述段落以"我"开头 | 我醒了。我走到镜子前。 | 他醒了。他走到镜子前。 |
| ❌ 整段用第一人称独白代替叙事 | 我今天不太对劲，我听见耳钉响了一下……（100 字后继续） | 改为第三人称白描 + 对白，内心用自由间接引语轻点 |
| ❌ 多个角色各占一段第一人称独白 | 第一段"林墨"视角 / 第二段"李医生"视角 | 所有人统一在第三人称叙事里，通过**动作 + 对白**呈现视角 |
| ❌ 角色独白标题分段（如 "林墨" / "李医生"） | `林墨\n\n6:40 我醒来。\n\n李医生\n\n我 08:15 坐下。` | 用一个叙述者声音贯穿，场景以"一/二/三"或空行切换 |

### 唯一合法的第一人称出现场景

- **对白引号内部**（角色说"我"是合法的）
- **自由间接引语**（极短，≤ 20 字，嵌入叙事段落中作为一个角色的思维闪过，不独立成段）
  ```
  ✅ 他看着镜子里的自己。没关系，他想，今天也在。
  ✅ 他把手伸进大衣口袋。空的。两年零十一个月，他想，病号服没有口袋。
  ❌ （独立成段）我没关系。今天也在。
  ```

### 为什么

Agent 回合制交互里每个 Agent 都是第一人称发言/反应——**这是素材不是成品**。编译者必须把 N 个第一人称素材**熔合成一个第三人称叙事**，让读者感受的是"一个冷静的叙述者在记录一个屋子里同时发生的事"，而不是"N 个人各自独白的拼图"。

这是剧场 vs 独白剧场的根本差别。

## 改写原则

- **忠于交互记录**：每个回合的台词/动作必须保留（可轻微改写节奏，不改变事实）
- **第三人称统一视角**：所有 Agent 的第一人称素材熔入同一个第三人称叙述者
- **对白直接引用**：引号内的对白尽量保留原话（最多做语气词的微调）
- **动作保留**：交互记录里的动作（"合上病历""手伸到半空"）必须出现在 novel.md 里
- **环境描写**：world-manager 的 broadcast 作为环境底色，可分散到场景不同位置
- **内心用自由间接引语**：不用"我"独立段，用"他想"/"他知道"/视角切换暗示内心
- **删繁就简**：交互记录里的冗余、过多内心独白、Agent 之间的客套都可砍掉
- **不新增事件**：interactions 里没有的关键事件不能补

## 🚫 去 AI 味规范（强制执行）

### 禁止清单

| 类别 | 禁止 | 替代方式 |
|------|------|----------|
| **破折号滥用** | 每段都出现"——"做解释或转折 | 全文破折号 ≤ 8处。用逗号、句号、换行替代 |
| **"不是X，是Y"句式** | "不是恐惧——是兴奋" 这类对偶 | 全文 ≤ 3处。改用具体描写或直接陈述 |
| **单词/短语独占一段** | "血。""等。""黑暗。" | 偶尔用（高潮处 ≤ 2次/千字），不能成为习惯 |
| **排比三连** | "填满了A。填满了B。填满了C。" | 全文排比 ≤ 2处。改用长句自然铺陈 |
| **对偶/对仗句** | "一个走向X，一个走向Y" | 全文对偶 ≤ 2处。用不对称的表达更自然 |
| **过度修辞** | 每段都有比喻/拟人 | 修辞要克制。一个场景最多 1-2 处比喻 |
| **解释性旁白** | "那是查克拉的压迫。无意识的。" | 让读者自己感受，不要解释 |
| **标记暴露** | `[inner_thought]`、`*[tag]*` | 内心戏直接用叙事体融入，不加技术标记 |

### 文风目标

写出来的东西要像**一个有风格的人类作者**写的：

1. **句式长短交错**：3-5个中长句后接一个短句，不要全部碎片化
2. **段落有呼吸**：叙事段落 3-6 句为主体，对话段落短，但不是所有段落都只有一句
3. **少解释多留白**：角色为什么这样做，不用旁白解释，让行为本身说明
4. **具体胜过抽象**：不说"某种更本质的动摇"，说"指尖在抖，镜片滑到鼻尖"
5. **拒绝对称美学**：现实不对称。两个人告别不需要结构完美的对仗
6. **内心戏融入动作**：想法、动作、对话交织在一起，不是三板斧

## 格式标准

```
标题格式：
  # EP{XX} — {标题}
  （不用 ## 分幕标题。用空行 + 场景首句暗示切换）

场景分隔：
  用一个空行即可。如果是大跨度切换（时空跳跃），用三个居中星号：
                          ＊ ＊ ＊

段落规范：
  - 叙事段落：3-6句，100-250字
  - 对话密集段：每人一段，说话动作紧跟引号
  - 内心戏：直接写进叙事，用微妙的视角切换暗示（不用斜体标记）
  - 避免超过300字的大段落

对话格式：
  中文双引号""包裹。说话人动作跟在引号后或前。
  连续对话超过4轮时必须插入动作/环境描写。
  不要每句话都"XX说"——用动作标签替代。

内心独白格式：
  ❌ *斜体独立段落*
  ❌ [inner_thought] 标记
  ✅ 融入叙事视角："他看着那双手，想起灭族之夜的月光。那双手曾经……"
  ✅ 自由间接引语：混在叙事中不加引号，用语气和人称暗示是角色在想

禁止出现：
  - Markdown 格式标记（##、**加粗**、---分隔线）
  - 技术性标签（[inner_thought]、[action] 等）
  - Emoji
  - "EP06 · 完" 之类的元标记
```

## 自检计数器（写完后验证）

交付 novel.md 前必须检查：

- [ ] **第三人称检查**（A 级硬约束）：叙述段落（对白引号外）无以"我"/"我们"/"咱"开头的独立句子
- [ ] **无角色名独白标题**（A 级硬约束）：无类似"林墨\n\n..."、"李医生\n\n..." 的第一人称分段头
- [ ] 破折号"——"出现次数 ≤ 8
- [ ] "不是X，是/而是Y"句式 ≤ 3
- [ ] 单词独占一段 ≤ 5处
- [ ] 排比三连 ≤ 2处
- [ ] 对偶/对仗 ≤ 2处
- [ ] 无 [inner_thought] 或 *斜体标记段*
- [ ] 无 ## 标题、--- 分隔线、**加粗**
- [ ] 段落最小单位 ≥ 2句（极个别高潮处例外）
