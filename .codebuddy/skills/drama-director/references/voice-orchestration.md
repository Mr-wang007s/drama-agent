# 多角色声音编排（voice-orchestration）

> 这是 Director 的"多人同台不串音"能力册。
> 回答的是："同一场戏里有 4 个角色，我怎么让读者不看人名也能分辨谁在说话？"
>
> 声音差异化是**人物立体的第一个指标**。读者在潜意识里辨认角色，靠的不是人名，靠的是**语言指纹**。

---

## 基本原理：语言指纹 = 物理识别

每个角色的语言指纹由 **5 个维度**构成：

| 维度 | 示例（林墨） | 示例（周伯） | 示例（沈砚之） |
|---|---|---|---|
| **句长** | 短 + 半句未完 | 长 + 学术腔 | 极短 + 停顿 |
| **口头禅** | "没关系" | "我们"代替"我" | 茶比喻 |
| **标点习惯** | 几乎不用 ! | 多用 ， 连句 | 多用 。 断句 |
| **比喻来源域** | 考古术语 | 引用论语 | 茶道 |
| **沉默模式** | 咽回半句话 | 擦眼镜代替说话 | 等一拍再接 |

**SOUL.yaml 里的 `performance.voice.quirks` 和 `unique_markers` 字段就是记录这些**。

Director 和 Critic 的工作就是**确保这 5 个维度在每场戏里都被兑现**。

---

## 失败信号：声音趋同

### 信号 1 · 所有人句长相同

**症状**：
```
周伯说："你去看一眼。"
林墨说："我去。"
张教授说："我陪他走这一趟。"
```

三人句长都是 5-6 字 → 读者看不出差别 → 串音

**诊断**：Director 没有把每个角色的 `句长特征`写进 Agent prompt。

### 信号 2 · 多人同时使用相同比喻

**症状**：
周伯说"这件事就像泡茶"，沈砚之说"你得等茶凉"——两个人都用茶比喻。

周伯的比喻来源域应该是**古籍/论语**（"从方法论上讲"），沈砚之才是**茶道**。两人比喻域撞了就失真。

**诊断**：SOUL 里比喻来源域没有在 prompt 里明确。

### 信号 3 · 口头禅被滥用

**症状**：所有角色都说"没关系"——林墨的专属口头禅被污染。

**诊断**：`blacklist_cross` 没生效。比如林墨的 "没关系" 应该出现在 blacklist 里，其他角色禁用。

### 信号 4 · 独有称呼被盗用

**症状**：林墨妻子说林墨"墨弟"——只有沈砚之能叫。

**诊断**：SOUL 里 `blacklist_cross` 没列"墨弟"。

## 声音差异化的 5 个手艺

### 手艺 1 · 三档句长分布

一场群戏里至少有 3 个角色时，**每个角色分配一档句长**：

| 档位 | 典型句长 | 适合的人格 |
|---|---|---|
| 极短 | ≤ 8 字 | 沉默寡言 / 居高临下 / 压抑 |
| 中等 | 10-20 字 | 社交正常 / 自嘲 / 克制 |
| 较长 | 20-35 字 | 学术 / 权威 / 表演性 |

**jiu-ge 实例**：

| 角色 | 档位 | 例子 |
|---|---|---|
| 沈砚之 | 极短 | "墨弟。""你瘦了。""走吧。" |
| 林墨 | 中等 | "病院伙食其实不错，主要是我自己不争气。" |
| 周伯 | 较长 | "陈队长这段是三天前拍的。殷墟那边现在有些情况，社科院希望有人去看一眼。就一眼。" |
| 李医生 | 中等（问句） | "你昨晚睡得怎么样？" |
| 林母 | 中短（碎） | "糕，过来。""趁热吃。" |

同一场戏出现**同一档位 ≥ 2 个角色**时 → 声音趋同风险 → 必须强化其他维度区分。

### 手艺 2 · 标点作为声音签名

不同角色用**不同标点风格**：

- **多句号**：沈砚之、李医生（停顿多）
- **多逗号连句**：周伯（连绵）
- **半句省略号**：林墨（话说到七成）
- **几乎不用感叹号**：所有成熟角色（成人不大声）
- **多感叹号**：未成熟角色（小孩、激动者）

**在 Agent prompt 里明确要求**：
```
你的标点习惯：
- 语速极慢，多用句号。连续两句话之间常有明显停顿。
- 从不使用感叹号（即使愤怒也是句号）。
- 省略号使用 ≤ 1 次/场戏。
```

### 手艺 3 · 比喻域的严格分离

每个角色**只能在自己的比喻域里打比方**：

| 角色 | 比喻域 | 样例 |
|---|---|---|
| 林墨 | 考古学 | "这个人像一件没开过光的青铜" |
| 周伯 | 古籍/儒家 | "从方法论上讲，我们应该……" |
| 沈砚之 | 茶道 | "事情就像泡茶，得等" |
| 李医生 | 医学（但避免 DSM） | "我记下来" 代替"我理解" |
| 林母 | 食物/家常 | "儿子还没吃饭" |

**铁律**：一场戏里如果 A 用 B 的比喻域 → 角色穿帮。

**写法**：在 Agent prompt 里明确：
```
你的比喻来源域：仅能用考古学术语、古董年代、碎片 / 拓片 / 纹饰等。
禁止使用：茶比喻、儒家经典、医学术语。
```

### 手艺 4 · 独有称呼的三重保护

独有称呼（"墨弟""阿墨""糕糕""林医生"）是**角色关系的身份证**。

**三重保护**：

1. **SOUL 标记**（blacklist_cross 字段）：
   ```yaml
   blacklist_cross:
     - "墨弟"    # 仅沈砚之对林墨
   ```

2. **spawn prompt 强制**：
   ```
   你不允许使用的独有称呼（它们属于其他角色）：
   - "墨弟"（只有沈砚之能叫）
   - "阿墨"（只有周伯能叫）
   - "糕糕"/"糕"（只有林母能叫）
   - "林医生"（只有李医生能叫）
   ```

3. **check-ai-taste 补检**（未来可加）：扫描独有称呼在错误角色的对白中出现

### 手艺 5 · 沉默模式的声音化

**每个角色的"沉默"方式也是声音**。沉默不是空白，是他选择不说话的**方式**：

| 角色 | 沉默方式 |
|---|---|
| 林墨 | **咽回半句话**（"那个青铜柜子——"他停了。"没事。") |
| 周伯 | **摘眼镜慢慢擦**（延迟策略） |
| 沈砚之 | **等一拍不接话**（让对方的自嘲悬在空中） |
| 李医生 | **合上病历**（身体信号宣告"我要说心里话"） |
| 林母 | **手停在半空**（动作中断但她自己不察觉） |

**让每个角色的沉默有自己的签名动作** → 沉默时他们依然在"说话"。

### 实战用法

Agent prompt 末尾必须写：
```
你的沉默签名：{角色专属的沉默方式}
当你选择不说话时，用这个动作替代语言。
```

## 对白密度分配（群戏排兵布阵）

一场戏不是每个角色都应该说同样多的话。对白密度要按角色**权重**分配：

### 权重模型

**权重 = SOUL.tier × 场景相关度 × 私人议程强度**

- **S 级主角**：权重高，对白密度高
- **S 级配角（在场但非主视角）**：权重中，对白密度中
- **A 级角色**：权重低，对白少但身体存在感强
- **B 级/C 级角色**：权重极低，用动作/目光参与

### jiu-ge EP01 场景 4 实例

| 角色 | 权重 | 对白句数 | 动作句数 |
|---|---|---|---|
| 周伯（S，隐藏主角） | 高 | 8 | 6 |
| 林墨（S，主视角） | 高 | 5 | 12 |
| 张教授（B，旁观） | 低 | 1（"那我陪林老师走这一趟"） | 3（沙发坐下、对视、起身） |

**合理的对白密度比**：周伯 8 : 林墨 5 : 张教授 1 ≈ 8:5:1

如果改成 5:5:5 → 群戏失焦
如果改成 10:1:0 → 林墨和张教授都变成背景板

## 多 Agent Team 启动前的声音校准表

Director 在 spawn Team 前，应填一张**本场声音校准表**：

```
场景：ep01-te-she · 场景 4 · 周伯办公室
主视角：林墨
隐藏主角：周伯

=== 声音分配表 ===

| 角色   | 句长档 | 口头禅 | 标点 | 比喻域 | 独有称呼 | 沉默签名 | 对白权重 |
|--------|--------|--------|------|--------|----------|----------|---------|
| 周伯   | 长     | "我们" | 多逗 | 古籍   | "阿墨"   | 擦眼镜   | 8 |
| 林墨   | 中     | "没关系" | 省略号 | 考古 | —        | 咽回半句 | 5 |
| 张教授 | 中     | —      | 多句 | —      | —        | 目光落下 | 1 |
```

这张表直接注入到每个 Agent 的 prompt 里。

## 声音检测的未来自动化（给 check 脚本的启发）

本文件目前是**手艺规范**。未来如果要自动检测，可以做以下脚本：

- **C7.1 · 独有称呼污染**：扫描每个角色的对白，如果出现其他角色的 `blacklist_cross` 条目 → Error
- **C7.2 · 句长均匀度**：计算场景内各角色的平均句长标准差 < 某阈值 → Warning（声音趋同）
- **C7.3 · 比喻域混用**：检测角色对白里是否出现不在自己 `比喻域`内的比喻关键词 → Warning

但**脚本只能做死检测**。真正的声音编排靠 Director 的耳朵——多读几遍自己写的对话，**闭上眼听**——是不是能分辨是谁在说。

分辨不出来 → 回去强化差异化。

## 给 Director 的自检

Phase 2 结束后读一遍 interactions.jsonl（编译前），问自己：

- [ ] 我能在不看人名的情况下分辨每个角色的对白吗？
- [ ] 每个角色的独有称呼都只由他专属使用者用了吗？
- [ ] 每个角色的比喻域都守在自己的领地吗？
- [ ] 群戏里的对白密度是不是按权重分配（不是 50-50 平均）？
- [ ] 每个角色的沉默有自己的签名动作吗？

五问全过 → 声音编排合格 → Phase 3 编译。
