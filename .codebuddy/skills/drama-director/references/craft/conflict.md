# 冲突学 · Conflict Design

> 故事的发动机。
> 本文件是**编剧**的核心手册，也是**导演**审 beat-sheet、**责编**诊断节奏的事实源。
> 核心命题：冲突不是"吵架"，冲突是"两种不兼容的欲望在同一个空间"。

---

## 一、冲突的本质定义

### 错误定义 vs 正确定义

```
❌ 冲突 = 角色在吵架 / 打架 / 对立
✅ 冲突 = 角色 A 想要的东西 必然 以 角色 B 付出某种代价 为前提
```

关键字：**必然**、**代价**。

没有代价 = 没有冲突。哪怕两个角色说得再激烈，只要其中任何一方"退一步"对自己没有代价，这就不是冲突——是假冲突。

### 检验标准

问一句话：**"如果 A 得到了 TA 想要的，B 会失去什么？"**

如果答案是"什么也不失去"——那这场戏没有冲突，是对话练习。

### 样例对比

```
❌ 假冲突：
  林墨：我想去殷墟。
  周文渊：现在不合适。
  林墨：为什么？
  周文渊：时机未到。
  （两人态度不同，但周文渊"让林墨去"对他没有代价——只是担心。这不是冲突。）

✅ 真冲突：
  林墨：我想去殷墟。
  周文渊：你去了，我就要把你父亲当年的卷宗给方士会看。
  林墨：你不敢。
  周文渊：我今天就可以。
  （周文渊必须在"保护林墨"和"保密卷宗"之间二选一——他付出的代价是失去一个活人或一个秘密。）
```

## 二、冲突四象限

任何场景冲突可以放进这个二维矩阵：

```
              外部冲突（Outer）    内部冲突（Inner）
  欲望层                                    
  （Want）    A 和 B 想要           A 同时想要
              同一个东西            两个不兼容的东西
              
  价值层                                    
  （Value）   A 和 B 世界观         A 的某个新事实
              不可调和              推翻了自己的价值观
```

### 四种典型冲突

**类型 1：外部欲望冲突（最常见，最易写）**
- A 想要 X，B 也想要 X，但 X 只有一个
- 样例：兄弟争遗产、情敌争爱人、敌对阵营争情报

**类型 2：外部价值冲突（最有深度）**
- A 和 B 对"什么是对"有根本分歧
- 样例：正义 vs 秩序、自由 vs 安全、个人 vs 集体

**类型 3：内部欲望冲突（最能塑造人物）**
- 角色自己同时想要两样不兼容的东西
- 样例：想复仇 vs 想原谅、想离开 vs 想留下
- **写好这一型 = 角色立住了**

**类型 4：内部价值冲突（最容易写成说教）**
- 角色的某个新事实推翻了自己的旧价值观
- 样例：发现父亲不是英雄、发现自己信的"真相"是假的
- 慎用——容易变成大段心理独白

### 单场景冲突配比原则

**每场戏必须包含至少 1 个外部冲突 + 1 个内部冲突。**

只有外部冲突 = 动作片
只有内部冲突 = 散文
两者叠加 = 故事

### 编剧实操：场景冲突矩阵表

beat-sheet v3 中每个场景必须列出：

```yaml
scene_3:
  outer_conflict:
    type: "外部价值冲突"
    A: "林墨"
    A_want: "知道父亲死亡真相"
    B: "周文渊"
    B_want: "保护林墨不被方士会盯上"
    clash: "知道真相 = 被盯上；被保护 = 永远不知道"
  
  inner_conflict:
    character: "林墨"
    want_1: "想相信周伯是为他好"
    want_2: "想冲出去找真相"
    tension: "每一次被周伯拦住，他都在心里把周伯往'共犯'的位置挪一寸"
```

## 三、三幕结构——故事的骨骼

### 亚里士多德版（经典）

```
第一幕（Setup）        25%    建立世界 + 主角 + 打破常态
第二幕（Confrontation）50%    主角遭遇阻碍 + 情节复杂化
第三幕（Resolution）   25%    冲突高潮 + 新常态建立
```

### 现代单集版（22-25 分钟电视剧 / 单集小说）

以一集 = 8000 字为例：

```
第一幕 · 2000 字（25%）
  - 场景 1：日常 / 上一集悬念延续
  - 场景 2：扰动事件（disturbance）——本集的"麻烦"来了
  - 转折点 1：主角不得不行动

第二幕 · 4000 字（50%）
  - 场景 3-4：主角尝试解决，失败
  - 场景 5：情境恶化 / 新信息颠覆
  - 转折点 2：一切都看起来要完了（暗点/midpoint）

第三幕 · 2000 字（25%）
  - 场景 6：主角用新方法/新觉悟应对
  - 场景 7：高潮对决
  - 场景 8：收束 + 埋新钩子
```

### 常见的三幕错误

**错误 1：第一幕太长**
- 症状：5000 字还在"介绍"，主角还没遇到麻烦
- 后果：读者弃文
- 修法：扰动事件必须在前 20% 出现，不管多仓促

**错误 2：第二幕注水**
- 症状：主角来回反复，没有情境升级
- 后果：读者"走神"
- 修法：每 1500 字必须有一次"情境升级"（新信息/新威胁/新盟友/新背叛）

**错误 3：第三幕仓促**
- 症状：前面铺 6000 字，最后 500 字把高潮和收束全写完
- 后果：读者觉得"为什么结束了"
- 修法：第三幕至少 1800 字，高潮 + 余韵缺一不可

## 四、Save the Cat 15 节拍表（Blake Snyder）

### 原版（电影 110 分钟）

Blake Snyder 在《救猫咪》里提出的 15 节拍表是好莱坞编剧最常用的模板。适配到**单集小说（8000 字）**如下：

| 节拍 | 英文 | 字数位置 | 功能 |
|---|---|---|---|
| 1. 开场画面 | Opening Image | 0-200 字 | 建立基调 / 暗示主题 |
| 2. 主题陈述 | Theme Stated | 200-500 字 | 配角无意中说出本集主题（主角还听不懂） |
| 3. 铺垫 | Set-Up | 500-1500 字 | 展示主角的"常态生活" + 三个需要改变的缺陷 |
| 4. 触发事件 | Catalyst | 1500-1800 字 | 打破常态的事件（信件/死讯/邂逅） |
| 5. 辩论 | Debate | 1800-2200 字 | 主角犹豫"要不要去" |
| 6. 进入二幕 | Break into 2 | 2200-2400 字 | 主角做出决定 / 被迫行动 |
| 7. B 故事 | B Story | 2400-2800 字 | 副线登场（通常是情感线 / 配角线） |
| 8. 乐趣与游戏 | Fun and Games | 2800-4200 字 | 电影海报上的那些画面 / 主角用新身份行动 |
| 9. 中点 | Midpoint | 4200-4400 字 | **虚假胜利** 或 **虚假失败** |
| 10. 坏人逼近 | Bad Guys Close In | 4400-5500 字 | 敌人反扑 + 主角内部矛盾激化 |
| 11. 一无所有 | All Is Lost | 5500-5800 字 | **死亡气息** 的瞬间（某人死/某物毁/某事败） |
| 12. 灵魂黑夜 | Dark Night of the Soul | 5800-6500 字 | 主角独处反思（本集情绪最低点） |
| 13. 进入三幕 | Break into 3 | 6500-6800 字 | 主角想通了 / 得到新工具 |
| 14. 结局 | Finale | 6800-7700 字 | 高潮 + 解决 |
| 15. 终场画面 | Final Image | 7700-8000 字 | 与开场画面对应（主题闭环） |

### 关键节拍详解

**节拍 9 · 中点（4200-4400 字，恰好 50%）**

这是整集最重要的位置。**必须发生一件"翻转"的事**：

- 主角以为要赢了 → 突然发现更深的陷阱（虚假胜利）
- 主角以为要输了 → 突然发现一线生机（虚假失败）

**中点没翻转 = 读者 50% 位置走神。** 这是新手最容易犯的错。

**节拍 11 · 一无所有（5500-5800 字）**

**必须有"死亡气息"**——物理死亡或象征死亡（关系死亡、希望死亡、身份死亡）。

> Blake Snyder 原话：读者需要"闻到"死亡的味道。

样例（林墨 EP03 在 5700 字处）：
> 林墨拨通了陈教授的电话。没人接。他又拨。第三次，一个陌生男声接了："陈教授昨晚去世了。"林墨没说话。他挂了电话。

这就是"一无所有"——陈教授不是林墨的家人，但他是林墨唯一可以验证父亲真相的老人。**象征性死亡**。

### 简化版：单集必备 5 节拍

如果 15 节拍太繁琐，底线是 5 个：

1. **触发事件**（1500-1800 字） — 打破常态
2. **进入二幕**（2200-2400 字） — 主角决定行动
3. **中点翻转**（4200-4400 字） — 一切颠倒
4. **一无所有**（5500-5800 字） — 死亡气息
5. **结局 + 新钩子**（7700-8000 字） — 解决本集 + 铺下集

**这 5 个节拍是底线。缺一个 = 本集节奏失败。**

## 五、7 节点模型（Dan Wells）

### 对比 Save the Cat

Dan Wells 的 7 节点模型比 Save the Cat 更适合**严肃/悬疑类**故事。Save the Cat 偏娱乐，7 节点偏文学。

### 7 个节点

```
节点 1 · Hook                  （开篇钩子）
节点 2 · Plot Turn 1           （情节转折 1：主角进入冒险）
节点 3 · Pinch 1               （压力点 1：外部阻力升级）
节点 4 · Midpoint              （中点：主角从被动变主动）
节点 5 · Pinch 2               （压力点 2：看起来要完了）
节点 6 · Plot Turn 2           （情节转折 2：主角得到最终工具/觉悟）
节点 7 · Resolution            （结局）
```

### 关键差异

**7 节点的 Midpoint 不是"翻转"，是"态度转变"。**

- Save the Cat 中点：**情境**翻转（好变坏/坏变好）
- 7 节点中点：**主角**翻转（被动变主动）

样例（林墨 EP04 中点）：
```
中点前：林墨一直在"被周伯带着走"——去哪、见谁、说什么，都是周伯安排
中点：林墨第一次拒绝——"我今晚不去方士会"
中点后：林墨开始自己查、自己找人、自己撒谎
```

这个"态度转变"才是 Dan Wells 说的 Midpoint。

### 选型建议

| 故事类型 | 推荐模板 |
|---|---|
| 爽文 / 网文 / 娱乐 | Save the Cat 15 节拍 |
| 悬疑 / 推理 / 严肃 | Dan Wells 7 节点 |
| 角色驱动文学 | 7 节点 + 强化中点态度转变 |
| 大型系列（单集内） | Save the Cat，但节拍适当压缩 |

## 六、反相位原则——主角与对手的节奏对位

### 什么是反相位

**反派/对手的行动节奏必须与主角相反。**

- 主角沉默时 → 对手出击
- 主角行动时 → 对手布局（不出击）
- 主角愤怒时 → 对手平静
- 主角虚弱时 → 对手强势
- 主角信任时 → 对手怀疑

### 为什么

相位相同 = 两个角色在做同一件事 = 冲突瓦解
反相位 = 两个角色的能量永远在撕扯 = 张力持续

### 样例（林墨 vs 周文渊 · EP02）

```
主角（林墨）节奏：            对手（周文渊）节奏：
场景 1：被动，听              主动，说 ← 反相位
场景 2：开始质问（出击）      沉默，点烟 ← 反相位
场景 3：愤怒拍桌              平静，推过来一杯水 ← 反相位
场景 4：崩溃，求答案          冷淡，看表 ← 反相位
场景 5：虚弱地坐下            突然起身，靠近 ← 反相位
场景 6：抬头看他              转身离开 ← 反相位
```

这种对位的戏**永远不会失败**。

### 编剧实操：反相位检查表

beat-sheet v3 中每个场景必须标注：

```yaml
scene_3:
  protagonist_state: "愤怒 + 出击"
  antagonist_state: "平静 + 防守"
  phase_match: "反相位 ✓"
```

如果发现`phase_match: "同相位"` → 这场戏要么删掉，要么改写其中一方的节奏。

## 七、反派设计——反派是"另一个主角"

### 核心原则

**反派不应该知道自己是反派。**

任何一个合格的反派，在他自己的叙事里，**他就是主角**。他有自己的创伤、动机、want/need/drive、秘密恐惧渴望。

### 反派的五个必备要素

1. **合理动机**：从反派视角看，他所做的事是对的
2. **能力匹配**：反派的能力必须≥主角（否则冲突瓦解）
3. **道德复杂性**：反派至少做过一件**读者会同情的事**
4. **与主角的镜像**：反派与主角有一个共同特质（创伤/童年/信念）
5. **不可妥协性**：反派的 Want 与主角的 Want 结构性互斥

### 周文渊的反派设计（样例）

```yaml
name: 周文渊
role: 卷一反角（但他自认为是林墨的守护者）

# 他自己的叙事视角：
self_narrative: |
  我父亲临终前把林家托付给我。林峰（林墨父亲）死后，我把林墨
  送进病院——这是救他不是害他。方士会要灭门，病院是唯一的遮蔽。
  三年后他出来了，我必须继续保护他——即使他恨我。

# 合理动机：
his_want: "让林墨活着"
his_need: "完成对死去老友的承诺，洗脱'见死不救'的内疚"
his_drive: "作为幸存者的过度补偿"

# 与主角的镜像：
mirror_with_protagonist: "都在为父亲活着（林墨为真相，周为承诺）"

# 道德复杂性（至少一件让读者同情的事）：
sympathy_scene: "EP02 中点，周文渊独处时看着林峰的照片说'我对不起你'"

# 不可妥协性：
why_clash_is_structural: |
  林墨想要真相；真相会让方士会发现林墨；方士会会杀林墨；
  周文渊阻止真相 = 阻止林墨死。
  两人都在"为林墨好"，但目标互斥。
```

**这样写出来的反派，读者不会骂"纸糊反派"。**

### 常见反派陷阱

**陷阱 1：为恶而恶**
- ❌ "他就是个坏人，喜欢看别人痛苦"
- ✅ "他做的每一件坏事，在他自己的逻辑里都是对的"

**陷阱 2：动机是"变态"**
- ❌ "童年受过虐待所以心理变态"
- ✅ 创伤链必须和主角一样完整——5 段因果

**陷阱 3：能力不足**
- ❌ 主角每次都能轻松挫败反派
- ✅ 反派至少胜主角 2 次，主角才能在第 3 次真正赢

**陷阱 4：没有反派的"人性时刻"**
- ❌ 反派永远冷酷
- ✅ 反派至少有一个场景让读者说"他也是人"

## 八、钩子经济学（入门·详见 mystery.md）

### 什么是钩子

**钩子 = 让读者想翻下一页/读下一集的未解之谜。**

### 钩子的三级

| 级别 | 定义 | 回收周期 | 密度（每集） |
|---|---|---|---|
| A 级 | 影响主线走向的核心谜题 | ≤ 5 集 | 1-2 个（每 2 集释放 1 个新 A） |
| B 级 | 影响支线/角色弧的悬念 | ≤ 10 集 | 2-3 个 |
| C 级 | 氛围性/伏笔型谜题 | 可 20+ 集 | 3-5 个 |

### 钩子经济的四个动作

**动作 1：释放**——埋下一个新谜题
**动作 2：强化**——让读者再一次想起这个谜题（但不给答案）
**动作 3：误导**——给一个看起来像答案但不是的解释
**动作 4：回收**——给出真正的答案（或下一层问题）

### 单集钩子预算

每集必须：
- 回收 ≥ 2 个前集钩子（至少 1 个 A 级）
- 释放 ≥ 2 个新钩子（至少 1 个 B 级）
- 强化 ≥ 1 个已有钩子（让读者记起来）

不达标 → 读者"不知道为什么要读下一集"。

### 更深入的钩子管理

见 `mystery.md` 第四节"钩子经济学"。

## 九、编剧的 8 问自检（原 writing-coach 迁移）

beat-sheet 写完后，编剧必须自问 8 个问题。任何一条回答不清晰 → 重写 beat-sheet。

| Q | 问题 | 判定 |
|---|---|---|
| Q1 | 本集核心冲突一句话说清（≤30 字，含"谁+想要什么+被什么阻碍"）？ | 必答 |
| Q2 | 主视角情绪起点和终点分别是什么（1-10 标注，必须有变化）？ | 必答 |
| Q3 | 爽点在哪个 Beat？为什么这里是爽点？ | 必答 |
| Q4 | 回收哪些前集钩子？释放哪些新钩子？（引用 hooks-ledger） | 必答 |
| Q5 | 有破防戏吗？满足 R1-R5 吗？（见 prose.md） | 有则答 |
| Q6 | 悬疑三铁律能满足吗？（见 mystery.md） | 悬疑类必答 |
| Q7 | 每个出场角色的"私人议程"是什么？ | 必答 |
| Q8 | 作为读者读完会想立刻看下一集吗？为什么？ | 必答 |

### 6 条红线（任中 1 条 → 重写 beat-sheet）

- R1：Q1 说不清一句话核心冲突
- R2：Q3 无法指出明确爽点
- R3：Q4 回收钩子 = 0
- R4：Q6 悬疑三铁律一条都不满足（悬疑类）
- R5：Q8 回答为"不会"或模棱两可
- R6：beat-sheet 字数预算总和 < 6000 中文字符

### 编剧的自检仪式

```
编剧 Agent 完成 beat-sheet 草稿后，必须：

1. 逐条写下 8 问的答案（≥3 句/条）
2. 对照 6 条红线自评
3. 如果任意一条红线触发：
   a. 标注触发的红线编号
   b. 说明如何修订
   c. 重写 beat-sheet
4. 再次自检，直到 0 红线

这 8 问的答案写入 beat-sheet 开头的 "编剧自检" 块。
```

## 十、冲突学检查清单（编剧+导演共用）

写完 beat-sheet 后过一遍：

### 结构层
- [ ] 第一幕在 25% 完成（触发事件出现）
- [ ] 中点（50%）有翻转/态度转变
- [ ] 一无所有（70-75%）有死亡气息
- [ ] 第三幕不少于总字数的 25%

### 冲突层
- [ ] 每场戏有至少 1 个外部冲突 + 1 个内部冲突
- [ ] 每场戏标注了冲突四象限类型
- [ ] 主角和对手的相位标注（反相位 ✓ / 同相位 ✗）
- [ ] 每场戏的"代价"清晰（A 得到 X 时 B 失去什么）

### 角色层
- [ ] 每个出场角色有三层动机（Want/Need/Drive）
- [ ] 反派（如本集有）在他自己视角里是主角
- [ ] 反派有一个让读者同情的瞬间

### 钩子层
- [ ] 回收 ≥ 2 个前集钩子（A 级 ≥ 1）
- [ ] 释放 ≥ 2 个新钩子（B 级 ≥ 1）
- [ ] 强化 ≥ 1 个已有未回收钩子

### 编剧自检层
- [ ] 8 问全部回答
- [ ] 6 条红线全部通过

**任何一项未勾 → 回 Phase 1 修订。**

## 十一、常见陷阱汇总

### 陷阱 1：冲突外化不足

```
❌ 两个角色的价值观对立，但全程只是对话，没有具体的对立事件
✅ 价值观对立必须通过"A 的选择让 B 失去了什么"这样的具体事件外化
```

### 陷阱 2：三幕比例失调

```
❌ 第一幕 4000 字 + 第二幕 3000 字 + 第三幕 1000 字
✅ 第一幕 2000 字 + 第二幕 4000 字 + 第三幕 2000 字
```

### 陷阱 3：中点没翻转

```
❌ 整集平稳进行，没有"突然一切变了"的瞬间
✅ 中点必须让读者"倒吸一口凉气"
```

### 陷阱 4：反派能力不足

```
❌ 主角智商吊打反派
✅ 反派至少让主角输 1-2 次，主角才能在最后真正赢
```

### 陷阱 5：钩子通胀

```
❌ 每集埋 10 个新钩子，回收 0 个 → 读者积压焦虑
✅ 每集埋 2-3 个 + 回收 2 个 + 强化 1 个 = 读者的好奇心被"喂饱"但不过载
```

### 陷阱 6：冲突收束过早

```
❌ 主角在 60% 位置就解决了本集核心冲突 → 后 40% 注水
✅ 核心冲突必须在 90% 位置解决，留 10% 余韵
```

## 附录 A：编剧的 Spawn Prompt 模板

```
你是编剧。你加载了以下 craft 文件：
- conflict.md（本文件）
- scene-design.md（场景学）
- mystery.md（类型学，如为悬疑故事）

你的任务是：

1. 阅读 episode-brief.md（导演给的选角+基调）
2. 写 beat-sheet v3（场景节拍 + 三层动机 + 钩子经济）
3. 完成后对着本文件第九节的 8 问自检
4. 触发任何一条红线 → 自己重写，直到 0 红线
5. 产出 beat-sheet.md 写入 episodes/<ep-id>/ 目录

beat-sheet v3 必含：
- 每场戏：字数预算 + 冲突四象限 + 反相位标注 + 三层动机
- 编剧自检块（8 问答案）
- 钩子经济表（回收/释放/强化）
- 前集事实核对清单（回指必查原文）

你不是导演。你不选角。你基于导演选好的角色写戏。
你不是责编。你不审稿。你基于知识写骨架。
```
