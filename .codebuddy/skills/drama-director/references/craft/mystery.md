# 悬疑学 · Mystery Craft

> 悬疑的本质是**信息管理**——什么时候给读者什么信息。
> 本文件是**悬疑顾问**的核心手册，也是**编剧**处理线索/钩子、**责编**审谜题密度的事实源。
> 核心命题：**悬疑不是藏真相，悬疑是让读者想知道。**

---

## 一、悬疑的本质

### 悬疑 vs 反转 vs 惊悚

这三个词经常被混用，但定义完全不同：

| 概念 | 定义 | 读者状态 |
|---|---|---|
| **悬念（Suspense）** | 读者看着事情发展，想知道结果 | "接下来会怎样？" |
| **悬疑（Mystery）** | 读者看着谜题，想找出真相 | "到底是什么？" |
| **反转（Twist）** | 读者以为 A，其实是 B | "居然是这样！" |
| **惊悚（Thriller）** | 读者感到角色被威胁 | "他会死吗？" |

**悬疑 = 读者和主角一起推理 → 读者希望自己先猜到**。

### Hitchcock 经典定义

> "炸弹在桌下。两个男人坐在桌上聊天。如果观众**不知道**炸弹——这叫惊讶。如果观众**知道**炸弹——这叫悬念。"

这段话的核心：**观众的信息量决定了戏的类型**。

- 观众 = 角色：惊讶（surprise）
- 观众 > 角色：悬念（suspense）
- 观众 < 角色：悬疑（mystery）——我们想知道角色知道什么

## 二、悬疑三铁律（每集必满足）

适用于 `.story.json` 中 `genre` 含 `mystery` / `thriller` / `crime` 的故事。
**每一条违反 = 本集悬疑属性失败。**

### 铁律 1：判断错 ≥ 1

**每集至少一处"主角/读者的合理推断被事实证伪"**。

#### 为什么

悬疑的核心动力是"信息不对称"。如果主角总是推断正确 → 读者失去推理乐趣 → 悬疑瓦解。

#### 判断错的三种类型

**类型 A：身份判断错**
- 以为 X 是坏人 → 其实是好人（或反之）
- 样例：以为周伯在保护林墨 → 其实他和方士会有协议

**类型 B：因果判断错**
- 以为 A 导致 B → 其实是 C 导致 B
- 样例：以为父亲死于心脏病 → 其实是被下毒

**类型 C：时间判断错**
- 以为事件 X 发生在时间 T1 → 其实发生在时间 T2
- 样例：以为信是昨晚送到 → 其实信一周前就在桌上

#### 检验标准

**如果删掉"判断错"场景，读者对真相的理解不变 → 这个判断错是假的。**

真正的判断错必须**影响后续情节**——错误的推断导致了错误的行动。

#### 编剧实操

beat-sheet v3 必须标注：

```yaml
misjudgment:
  who: "林墨"
  what_he_thinks: "周伯是父亲的老友，值得信任"
  reality: "周伯和方士会有过协议——三年前他主动提议把林墨送进病院"
  consequence: "林墨在第一次对质时毫无防备，信息处于不对称"
  reveal_episode: "EP07"  # 什么时候证伪
```

### 铁律 2：第三方物理反应 ≥ 1

**每集至少一处"不在场角色的行为间接证明/否定某事"**。

#### 为什么

直接告知的信息读者不会记住。**通过物理线索间接获得的信息读者会反复思考**。

#### 第三方物理反应的 7 种实现

**实现 1：物品的异常**
- 桌上多了个杯子（说明来过人）
- 门锁方向反了（有人进来过又出去了）
- 椅子上有压痕但没有人

**实现 2：环境的残留**
- 烟灰缸里的烟灰还在冒（有人刚走）
- 地上的水渍形状（说明是站着看的方向）
- 空气里的香水味（不是主角的）

**实现 3：声音的缺席**
- 应该响的电话没响（已被接了/拔了线）
- 应该来的人没来（出事了）
- 应该有的背景音不在（有人关掉了）

**实现 4：NPC 的异常行为**
- 保洁阿姨突然不来这层楼（被调走）
- 保安看了主角一眼然后立刻低头（被嘱咐过）
- 小区邻居的狗不叫了（熟悉这个陌生人）

**实现 5：文件/记录的矛盾**
- 账本上的日期不对
- 照片里的人比实际少一个
- 文件边缘的指纹多于声称经手的人数

**实现 6：身体的不合逻辑**
- 某人的咳嗽太刻意
- 某人的呼吸节奏和说话不配
- 某人的汗出现在不该出现的时间

**实现 7：数字/细节的异常**
- 账单上的数字是整数（疑似编造）
- 时间点太精确（背过）或太模糊（掩盖）

#### 编剧实操

beat-sheet v3 中每场戏应标注第三方物理反应：

```yaml
scene_4:
  third_party_physical:
    type: "实现 1：物品的异常"
    detail: "周伯桌上的茶杯口朝向他自己——但林墨进门时他本应面朝门口"
    implies: "周伯在林墨进门前刚转过身藏了什么"
    reader_catches: "细心读者会注意到"
    character_catches: "林墨 70% 位置才意识到"
```

### 铁律 3：A 级钩子 ≥ 1（每 2 集释放 1 个新 A 级）

**A 级钩子 = 影响主线走向的谜题/悬念/威胁。**

- 每 2 集必须释放 1 个新 A 级钩子
- 奇数集推荐释放新 A 级，偶数集可以只回收

#### A 级钩子的标准

```yaml
A_class_hook_criteria:
  - 回答这个问题 = 推进主线
  - 不回答 = 后续章节无法进行
  - 读者能用一句话复述（不抽象）
  - 有时间压力感（多久必须揭晓）
```

不满足 → 是 B 级或 C 级钩子。

## 三、线索三明治

### 什么是线索三明治

**关键线索 = 被 2 个无关信息夹住的中间那一层。**

```
[无关信息 A]  ← 读者以为这是线索（误导）
[关键线索]    ← 真正的线索（被藏）
[无关信息 B]  ← 读者想这大概是线索（误导）

结构：无关 → 关键 → 无关
读者自然注意到的：2 个无关信息
读者自然忽略的：关键线索
```

### 为什么有效

读者阅读时会**预期"作者会强调线索"**。如果线索被"夹心"了，读者会：
- 注意到面包（无关信息）
- 忽略馅儿（关键线索）
- 等到后面真相揭露时"啊！原来那句是线索！"

**这个"啊"就是悬疑阅读的最大快感。**

### 样例

```
❌ 直白放线索：
  周伯从抽屉里拿出一块表。"这是你父亲的。"他把表递给林墨。

↑ 读者立刻知道"这块表重要"——但读者没有"自己发现"的快感。

✅ 线索三明治：
  周伯从抽屉里拿出一摞文件。"这些都是你父亲的东西。"他翻了翻——
  一个信封、一本 2019 年的日历、一块表、几张收据，还有那枚 
  他送给林墨的钢笔。周伯把信封和日历摊开在桌上。
  
  "信你自己看。日历里有他最后一个月的行程。"
  林墨接过来。他的视线在信封上停了一下，又移到日历上。

↑ 真正的关键是"表"——但它被放在文件清单的中间，周伯强调的是信和日历。
   读者会跟着周伯的视线注意信和日历。
   等第 N 集揭露"表里有秘密"时，读者会翻回来说"原来早就出现过"。
```

### 线索三明治的三种变体

**变体 1：物品清单型**（上面的例子）
- 关键物品混在列表中
- 读者注意列表的首尾

**变体 2：对话埋线型**
- 关键台词混在闲聊中
- 读者注意对话的开头和收尾

**变体 3：场景路过型**
- 关键场景被前后两个"重要场景"夹住
- 读者注意前后，忽略中间

### 编剧实操

每个 A 级钩子的"引入"场景必须用三明治结构。beat-sheet v3 标注：

```yaml
scene_3:
  hook_introduction:
    hook_id: H15
    level: A
    hook_content: "父亲的手表是倒着戴的——为什么？"
    sandwich_structure:
      bread_1: "信封（读者会以为这是线索）"
      meat: "手表（真正的线索，只用 1 句话带过）"
      bread_2: "日历（读者会记住日期）"
```

## 四、钩子经济学（完整版）

### 钩子的三级定义

| 级别 | 定义 | 回收周期 | 单集密度 |
|---|---|---|---|
| **A 级** | 影响主线走向的核心谜题 | ≤ 5 集 | 1-2 个（每 2 集释放 1 个新 A） |
| **B 级** | 影响支线/角色弧的悬念 | ≤ 10 集 | 2-3 个 |
| **C 级** | 氛围性/伏笔型谜题 | 可 20+ 集 | 3-5 个 |

### 钩子的四个动作

**动作 1：释放（Release）**
- 首次埋下谜题
- 必须让读者"察觉到"（否则无效）
- 但不能让读者"立刻知道全貌"（否则过早）

**动作 2：强化（Reinforce）**
- 让读者再次想起这个谜题
- 给一点新信息，但不给答案
- **每 A 级钩子在生命周期内至少强化 1 次**

**动作 3：误导（Misdirect）**
- 给一个看起来像答案但不是的解释
- 迫使读者修正自己的推理
- 重要：误导必须合理（不是瞎编的）

**动作 4：回收（Resolve）**
- 给出真正的答案（或揭开下一层问题）
- 回收时机必须有戏剧价值（不是"到点了该解答"）

### 钩子台账管理

每个故事必须维护 `world/hooks-ledger.md`（由编剧在 Phase 2 维护，不再用脚本自动化）：

```markdown
# Hooks Ledger · stories/<name>

## 当前活跃钩子

### A 级（≤5 集回收）

#### H03 · 父亲的真正死因
- 释放：EP01 L123-L145
- 强化：EP03 L456（验尸报告的异常）
- 误导：EP05 L789（周伯说"心脏病"）
- 当前状态：未回收
- 集龄：4 集（警戒线：5 集）
- 计划回收：EP07

#### H07 · 周伯的真实立场
- 释放：EP02 L234
- 强化：EP04 L567（手表倒戴）
- 当前状态：未回收
- 集龄：3 集

### B 级（≤10 集回收）

#### H15 · 林墨在病院第二年的失忆
- 释放：EP01 L89
- 当前状态：未回收
- 集龄：4 集

### C 级（可长期保留）

#### H22 · 殷墟出土的青铜器文字
- 释放：EP04 L123
- 当前状态：未回收
- 集龄：1 集（C 级不限期）

## 已回收钩子

| 编号 | 级别 | 释放 | 回收 | 回收方式 |
|---|---|---|---|---|
| H01 | A | EP01 | EP03 | 揭露父亲不是自然死亡 |
| H02 | B | EP01 | EP02 | 神秘电话是陈教授打的 |
```

### 钩子通胀的危险

**症状**：每集埋 10 个新钩子，回收 0 个 → 积压焦虑

**后果**：读者记不住，开始"不在乎"，悬疑张力崩塌

**防治**：单集钩子预算严格控制

```yaml
single_episode_hook_budget:
  new_release: 2-3 个
  reinforce: 1-2 个
  resolve: 1-2 个
  total_interaction: 4-7 次钩子事件
```

超过 8 个 = 通胀，减少下集的新钩子。

### 钩子密度不足的危险

**症状**：连续 2 集没有新 A 级钩子 → 读者"不知道为什么要追"

**防治**：每 2 集必须有 1 个新 A 级钩子。奇数集强制释放。

## 五、信息差管理节奏

### 信息差的四个阶段

一个核心谜题在故事中的信息差演变：

```
阶段 1：读者=主角=角色    （故事刚开始，所有人信息一致）
     ↓
阶段 2：主角发现异常       （读者与主角同步怀疑）
     ↓
阶段 3：信息不对称激化     （读者 > 主角 或 主角 > 读者）
     ↓
阶段 4：信息对称化         （谜题揭晓，回到阶段 1）
```

### 不同类型故事的信息差节奏

**类型 A：推理型（Whodunit）**
```
阶段 2 持续 70%（读者和主角一起猜）
阶段 3 持续 20%（主角突然明白，读者滞后）
阶段 4 持续 10%（揭晓）

读者快感：和主角一起推理
```

**类型 B：心理惊悚型**
```
阶段 2 持续 30%
阶段 3（读者>主角）持续 50%（读者替主角担心）
阶段 4 持续 20%

读者快感：看着主角走向陷阱
```

**类型 C：悬念型（Suspense）**
```
阶段 2 持续 20%
阶段 3（读者=主角）持续 60%（都知道危险即将来临）
阶段 4 持续 20%（知道炸弹何时爆）

读者快感：紧张感的持续累积
```

### 编剧实操

每个故事在 bible.md 应标注主信息差类型，每集在 beat-sheet v3 标注本集信息差：

```yaml
episode_information_asymmetry:
  primary_hook: H03
  stage: 3
  direction: "读者 > 主角"
  tension_source: "读者知道周伯和方士会有协议，主角还在相信他"
  planned_to_next_stage: "EP07 时进入阶段 4（揭晓）"
```

## 六、误导的艺术（Red Herring）

### 什么是误导

**Red Herring = 看起来像线索但不是线索的东西。**

名字来自英国猎狐传统——用红鲱鱼的气味干扰猎犬的嗅觉。

### 好误导 vs 坏误导

**好误导**：
- 在当时场景下，读者的推断是**合理的**
- 揭晓后，读者能接受"我被误导了"而不是"被作者骗了"
- 误导本身是故事信息的一部分（不是凭空造出来的）

**坏误导**：
- 作者刻意隐藏关键信息（不是读者推理失误，是作者没给信息）
- 揭晓后读者说"你凭什么这么设定"
- 误导是为了拖延揭晓

### 公平竞争原则（Fair Play）

推理小说的黄金法则（Ronald Knox 十诫）：

1. 所有线索必须在揭晓前已给读者
2. 不能靠巧合或超自然力解决
3. 不能有读者不可能推断的秘密
4. 凶手不能是最后一章才出场的人
5. 不能有未被描述的密道/密室机关
6. 不能靠侦探的"直觉"
7. 侦探本人不能是凶手（除非提前说明）
8. 关键线索不能被侦探找到后不告诉读者
9. 华生型角色不能对读者隐藏重要想法
10. 不能有双胞胎突然出现（除非提前铺垫）

### 误导的三个技巧

**技巧 1：巧合中的必然**
- 一个"巧合"发生两次 = 不是巧合
- 让真线索看起来像"又一次巧合"

**技巧 2：强调反方向**
- 用大段笔墨描写 A 方向
- 真相在 B 方向
- 但 B 方向的细节是**完整的**，不是作弊

**技巧 3：权威证言的错误**
- 让可信的角色（医生/警察/长辈）说错
- 他们不是故意撒谎——他们是被误导了
- 这也要求读者"不能完全信任角色的判断"

## 七、读者推理的三种参与度

### 三型读者

**Type 1：被动型（70% 的读者）**
- 不主动推理
- 跟着主角的节奏走
- 喜欢被意外
- 适合：Save the Cat 式反转节奏

**Type 2：同步型（25% 的读者）**
- 和主角同步推理
- 如果能先主角一步猜到会满足
- 喜欢"看作者怎么继续"
- 适合：推理小说标准节奏

**Type 3：主动型（5% 的读者）**
- 试图比主角先猜到
- 会回翻检查线索
- 喜欢"被挑战"
- 适合：公平竞争型推理

### 多层读者满足

一个优秀的悬疑作品同时满足三型：

- **表层**：情节悬念推动，被动读者跟得走
- **中层**：每个信息都"有用"，同步读者能跟着推理
- **深层**：所有线索"公平"，主动读者能回翻验证

### 编剧实操

写每个章节时，问自己三个问题：

1. 被动读者：TA 能跟上情节吗？
2. 同步读者：TA 手里有足够的线索吗？
3. 主动读者：TA 如果回翻会找到足够证据吗？

三者皆是 → 高质量悬疑。

## 八、悬疑顾问的工作流

### Phase 2 的诊断清单

悬疑顾问在 Phase 2 拿到 beat-sheet v3 草稿后，执行以下检查：

```yaml
mystery_advisor_checklist:
  三铁律:
    - [ ] 判断错 ≥ 1（指出具体场景）
    - [ ] 第三方物理反应 ≥ 1（指出具体物品/声音/环境）
    - [ ] A 级钩子释放 ≥ 1（如奇数集）or 回收 ≥ 1（如偶数集）
  
  钩子经济:
    - [ ] 新释放 2-3 个
    - [ ] 强化 1-2 个
    - [ ] 回收 1-2 个
    - [ ] 总交互 4-7 次
    - [ ] 没有通胀（>8 次）
  
  线索三明治:
    - [ ] 每个新 A 级钩子用三明治引入（无关→关键→无关）
    - [ ] 关键线索没有被"作者强调"
  
  信息差:
    - [ ] 本集主信息差类型标注清楚
    - [ ] 阶段推进符合故事类型的节奏
  
  公平竞争:
    - [ ] 没有作者刻意隐藏的关键信息
    - [ ] 所有线索在揭晓前都给了读者
```

### Phase 4 的复审

责编在 Phase 4 审完稿后，如果涉及悬疑环节，再 spawn 悬疑顾问做一次专题复审：

```
悬疑顾问 Phase 4 复审重点：
1. 线索在 novel.md 正文里是否真的被"埋"了（不只是 beat-sheet 上标注）
```

## 九、火影案例——鼬一族真相的悬疑结构拆解

火影疾风传中"宇智波鼬其实是忠于木叶的卧底"这个反转是一个教科书级的悬疑结构。拆解如下：

### 信息差节奏

```
第一集 - 第 200 集：读者 = 主角 = 角色
  所有人都以为鼬是屠族凶手

第 200 集 - 第 350 集：主角 > 读者（微弱）
  佐助开始怀疑，但读者还不确定

第 350 集 - 第 475 集：读者 > 主角
  读者通过各种蛛丝马迹意识到鼬的"自我污名化"
  主角还在想复仇

第 475 集：读者 = 主角 = 角色
  揭晓——鼬是被迫的
```

### 三铁律的应用

**判断错**：
- 所有人（主角+读者+角色）在 400+ 集里判断鼬是恶人
- 证伪：EP475 揭晓真相
- 这个"大判断错"影响了整个系列的故事走向

**第三方物理反应**：
- 鼬对佐助的每一个"冷酷"动作都有一个"关心"的微小身体信号
- 例如杀了佐助的好友后，手指会停顿 0.5 秒
- 读者回翻时会说"啊，原来这些都是线索"

**A 级钩子**：
- "鼬为什么屠族" → 持续 400 集的主线钩子
- 中间多次强化（鼬说话时的异常、小南的暗示）
- 最终回收满足三型读者

### 线索三明治

鼬的真相在多次"三明治"中被埋：

- 面包 1：鼬的残忍行为（大量）
- 关键线索：鼬对佐助的保护（少量、低调）
- 面包 2：其他反派的行为（同步发生）

读者第一次读时只看到"鼬是反派"，回翻时才看到"线索一直在"。

### 公平竞争

符合 Fair Play 原则：
- 所有线索在揭晓前都给了读者
- 没有突然冒出新设定
- 鼬的动机是完整的（他的创伤链和价值观可以支撑他的行为）

**这就是顶级悬疑结构。**

## 十一、常见陷阱

### 陷阱 1：作者替读者"指认"线索

```
❌ "林墨注意到周伯的手表戴反了——这一定有什么原因。"
```

↑ 作者在强调线索。读者失去推理乐趣。

```
✅ 林墨点头，把茶杯放下。周伯的手表贴在桌面上反着。林墨没说话。
```

↑ 线索一笔带过，读者自己决定要不要记住。

### 陷阱 2：钩子没有时间压力

```
❌ "父亲的死因一直是个谜。"
✅ "父亲的死因必须在七月之前查清——那是他忌日，有些人约好了那天做一件事。"
```

时间压力让读者有"必须继续看"的动力。

### 陷阱 3：回收时突然冒出新设定

```
❌ 揭晓：原来周伯是林墨的生父。（之前从未铺垫）
✅ 揭晓：周伯在 EP02 时摸了林墨的头发——那是长辈才会做的动作。（有铺垫）
```

### 陷阱 4：三铁律"打卡式"完成

机械地"补一个判断错 + 一个物理反应 + 一个 A 级钩子" → 三铁律只是为了合规。

真正的三铁律必须**有机融入**——判断错推动情节、物理反应是信息渠道、A 级钩子驱动主线。

### 陷阱 5：线索密度失衡

```
❌ 前三集零钩子 → 第四集埋 8 个钩子
✅ 每集稳定 2-3 个新钩子 + 1-2 个回收
```

### 陷阱 6：误导不公平

```
❌ "凶手是一个之前从未出现的神秘人。"
✅ "凶手是 EP01 就出现过的那个角色——但他当时以另一个身份出现。"
```

## 附录：悬疑顾问 Spawn Prompt 模板

```
你是悬疑顾问。你加载了 mystery.md（本文件）。

你的职责分两个阶段：

## Phase 2 诊断（编剧写完 beat-sheet 后）

对照本文件第十节清单，逐项打勾：

1. 三铁律
   - 指出"判断错"发生在哪个 beat
   - 指出"第三方物理反应"具体是什么（物品/声音/NPC）
   - 指出 A 级钩子的释放/回收

2. 钩子经济
   - 列出本集所有钩子事件（释放/强化/误导/回收）
   - 检查总数是否在 4-7 之间

3. 三明治结构
   - 新 A 级钩子的引入是否用了三明治
   - 如果没有，给出重写建议

4. 信息差
   - 标注本集主信息差类型和阶段

输出：诊断报告 + 修订建议（不直接改 beat-sheet，由编剧采纳）

## Phase 4 复审（novel.md 编译完成后）

重点审查：
- 线索在正文中的"埋藏质量"（是否被过度强调）
- 三明治结构是否在编译中被破坏
- 悬念感是否足够（读者会追下集吗）
- 公平性（读者会不会觉得被骗）

输出：悬疑维度审查意见，并入 editor-review.md
```
