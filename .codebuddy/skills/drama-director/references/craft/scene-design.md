# 场景学 · Scene Design

> 场景是故事的原子单位。
> 本文件是**编剧**设计场景、**表演指导**判断转折点、**责编**审节奏的共用手册。
> 核心命题：**一个好场景 = 一个微型故事。** 每个场景都有自己的开头-发展-高潮-收束。

---

## 一、场景的定义

### 什么不是场景

```
❌ 场景 = 一段时间+一个地点
❌ 场景 = 角色之间的一段对话
❌ 场景 = 作者想让读者看到的一段情节
```

### 什么是场景

```
✅ 场景 = 一次"状态改变"的完整过程

状态改变 = 进入场景时 vs 离开场景时，某样东西必须不同：
  - 关系变了（A 和 B 的信任度变了）
  - 信息变了（某人知道了之前不知道的事）
  - 决定变了（某人做了一个不可逆的选择）
  - 情感变了（某人的情绪从 X 到 Y）
  - 权力变了（谁在控制局面变了）
```

### 检验标准

问一句话：**"如果删掉这个场景，故事还能继续吗？"**

- 如果答案是"能"——这个场景不是必要的，删掉
- 如果答案是"不能，因为 XX 变化没有发生"——这就是必要场景

## 二、场景功能 8 分类

每个场景必须被编剧明确标注其**主功能**。8 种典型功能：

| 代号 | 功能 | 作用 | 推荐长度 |
|---|---|---|---|
| **F1 · 建立** | Establish | 建立世界/人物/关系的新状态 | 1200-1800 字 |
| **F2 · 推进** | Advance | 推动主线情节一步 | 1500-2000 字 |
| **F3 · 揭示** | Reveal | 揭露一个新信息/真相 | 1000-1500 字 |
| **F4 · 反转** | Reverse | 颠覆读者/角色的认知 | 1200-1800 字 |
| **F5 · 沉淀** | Reflect | 角色内化前面发生的事 | 800-1200 字 |
| **F6 · 过渡** | Transition | 连接两个大场景 | 400-800 字 |
| **F7 · 爆发** | Explode | 情感/冲突的高点 | 1500-2500 字 |
| **F8 · 收束** | Close | 本集收尾+新钩子 | 800-1200 字 |

### 单集功能配比（标准模板）

8000 字单集推荐配比：

```
场景 1：F1 建立      1500 字   本集起点
场景 2：F2 推进      1800 字   第一个事件
场景 3：F3 揭示      1200 字   第一个秘密
场景 4：F4 反转      1500 字   中点翻转
场景 5：F7 爆发      1800 字   情感高点
场景 6：F5 沉淀      1000 字   主角消化
场景 7：F8 收束      1200 字   收尾 + 新钩子
```

**关键原则：连续两场不能同功能。**

连续两个 F2 推进 = 读者觉得"又在走剧情"
连续两个 F5 沉淀 = 读者觉得"一直在想"

### 功能链的情绪波形

好的场景功能链应形成情绪波浪：

```
F1 平 → F2 小升 → F3 突升 → F4 翻转 → F7 大升 → F5 大降 → F8 微升
       ↑              ↑                   ↑              ↑
      起波         第一个爽点            高潮           新钩子
```

画出来像心电图。**平直的心电图 = 读者死了。**

## 三、场景进入原则——迟入早出（Late In, Early Out）

### 原则

**进入场景时，已经发生了一半。**
**离开场景时，问题还没解决。**

这是 Robert McKee 在《故事》里反复强调的铁律。

### 反例 vs 正例

**反例（早入晚出 / 新手写法）：**

```
林墨走到茶馆门口。他推开门。茶馆里有一些客人。他找了一张靠窗的桌子
坐下。服务员过来问他喝什么。他说要一壶茉莉花茶。服务员走了。他打
量四周，看到周文渊坐在角落里。他起身走过去。他坐到周文渊对面。

"你来了。"周文渊说。
"嗯。"林墨说。
...
（300 字后才进入真正的戏）
```

**正例（迟入早出）：**

```
"你来了。"周文渊已经等了四十分钟。
林墨在他对面坐下。茉莉花茶是周文渊替他点的——三年前林墨父亲
的忌日，周文渊也给他点过这个茶。
林墨看着茶杯，没抬头。

"你父亲的遗物里，有一封信。"周文渊说。
```

差别：

- 反例用 200 字"安排角色进入场景"
- 正例第一句就是戏的中段——周伯等了 40 分钟（这本身就是人物状态）
- 正例的"茉莉花茶"一笔带过历史（迟入但信息密度高）

### 操作规则

**进入场景**：
- ❌ 不写"到达" / "开门" / "坐下" / "打招呼"这类过场动作
- ✅ 直接从**某人说某句关键话** / **某个动作正在进行**开始

**离开场景**：
- ❌ 不写"最后他们结束了对话"这类收束句
- ✅ 在**某个动作戛然而止** / **某句话被打断**处切断
- ✅ 下一个场景开头用读者能自己推断的方式暗示"时间过去了"

### 过渡写法

两场戏之间不需要"过渡段"。方法：

```
[场景 A 结尾]
  周文渊起身走向门口，没回头。
  林墨看着他的背影——那背影和三年前父亲出殡那天一模一样。

[空一行]

[场景 B 开头]
  凌晨三点，林墨第七次拨通了陈教授的电话。
```

**一个空行 = 时空跳跃。读者会自己接上。**

## 四、场景转折点——黄金位置 60-75%

### 什么是场景转折点

场景内部也有三幕结构：

```
场景开头（0-30%）     场景发展（30-70%）     场景高潮（70-90%）    场景收束（90-100%）
铺垫 / 建立状态        推进 / 制造张力         转折点 / 反转          余韵 / 切出
```

**转折点 = 场景内部的"翻转时刻"**。位置在 60-75%。

### 转折点的三种类型

**类型 1：信息转折**——读者/角色获得一个颠覆性信息

```
场景前半：林墨以为周伯在帮他
场景 65%：林墨发现周伯桌上的文件——是和方士会签的协议
场景后半：林墨的每一句话都变成了试探
```

**类型 2：情感转折**——角色的情绪/态度突变

```
场景前半：林墨冷漠地应付周伯的问候
场景 70%：周伯说了一句"你父亲最后一天，一直叫你的名字"
场景后半：林墨的冷漠瓦解了，他开始主动问问题
```

**类型 3：权力转折**——谁在主导对话变了

```
场景前半：周伯主导——他问林墨答
场景 60%：林墨突然反问一句："你怎么知道那天在医院的是我？"
场景后半：林墨主导——周伯开始回避
```

### 编剧实操：转折点标注

beat-sheet v3 中每个场景必须标注转折点的**位置+类型+内容**：

```yaml
scene_4:
  budget_chars: 1500
  turning_point:
    position: "65%（大约第 975 字）"
    type: "权力转折"
    trigger: "林墨发现周伯手表戴反"
    before: "周伯主导——追问林墨病院生活"
    after: "林墨主导——周伯开始找借口"
```

### 转折点缺失的后果

症状：场景"平"——从头到尾一个情绪、一个节奏、一个权力结构。

后果：读者在这一场景走神。

修法：加一个转折点——哪怕小到"主角突然看见了某样东西"。

## 五、信息差三模型

### 什么是信息差

场景中的三方——读者、主角、其他角色——**知道的信息不一样**。

好的场景利用信息差制造张力；差的场景让所有人"知道一样多"。

### 模型 1：读者 > 主角（戏剧反讽）

**读者知道某事，主角不知道。**

用途：制造紧张感（读者替主角捏汗）、悲剧感（读者看着主角走向悲剧）

样例：
```
读者知道：周伯的书房抽屉里有林墨父亲的遗书
主角不知道：林墨以为所有遗物都已交给他

场景张力：林墨每一次靠近那张桌子，读者都在喊"找书！找书！"
```

**Hitchcock 经典论述**："炸弹在桌下，观众知道但角色不知道——这叫悬念。"

### 模型 2：主角 > 读者（保留悬念）

**主角知道某事，读者不知道。**

用途：制造悬念（读者想知道主角在想什么）、塑造深度（角色有秘密）

样例：
```
读者不知道：林墨三年前见过周伯一面（在病院黑市）
主角知道：但他装作不认识

场景张力：读者会注意到林墨的"过度冷静"——为什么？
```

**慎用**——超过 3 场戏不解密 = 读者放弃追踪。

### 模型 3：角色之间的信息差

**角色 A 知道，角色 B 不知道。**

用途：制造对话张力、让读者"替角色代入"

样例：
```
林墨知道：父亲的遗书存在
周伯知道：林墨已经找到了遗书
但两人都假装不知道对方知道

每一句对话都是双重含义的——这叫"潜台词场景"
```

这种场景是**对话剧的最高形态**。

### 单场景信息差配置原则

**每个场景至少配置一个明确的信息差方向。**

beat-sheet v3 必须标注：

```yaml
scene_5:
  information_asymmetry:
    primary: "读者 > 主角"
    detail: "读者知道周伯手表是倒戴的——林墨没注意到"
    intended_tension: "读者会一直盯着手表，等林墨发现"
```

### 信息差切换

**同一场景可以切换信息差模型**——这会制造二次反转：

```
场景前半：读者 > 主角（读者知道周伯在骗林墨）
场景 60%：周伯说了一句关键话，林墨意识到——切换为 角色对等
场景 80%：林墨假装没听出——切换为 主角 > 读者（读者不知道林墨已经警觉）
```

三段切换 = 读者全程屏住呼吸。

## 六、场景间的情绪落差

### 原则

**高能场景之后必须接低能场景。**

连续 2 个高能场景 = 读者审美疲劳
连续 2 个低能场景 = 读者走神

### 情绪能级标定

给每场戏标一个情绪能级（1-10）：

| 能级 | 特征 |
|---|---|
| 1-2 | 独处 / 日常 / 沉思 |
| 3-4 | 试探性对话 / 小信息交换 |
| 5-6 | 有实质冲突的对话 |
| 7-8 | 激烈冲突 / 重大揭示 |
| 9-10 | 破防 / 死亡气息 / 大反转 |

### 单集推荐波形

```
Scene 1: 3   (日常起点)
Scene 2: 5   (扰动事件)
Scene 3: 4   (试探)       ← 回落
Scene 4: 7   (中点翻转)   ← 爬升
Scene 5: 3   (沉淀)       ← 大幅回落（必要）
Scene 6: 9   (爆发)       ← 冲到顶
Scene 7: 5   (收束)       ← 回落 + 新钩子

波形：3 → 5 → 4 → 7 → 3 → 9 → 5
     （有起有伏，像山脉）
```

**错误波形**：

```
Scene 1: 5
Scene 2: 6
Scene 3: 7
Scene 4: 7
Scene 5: 8
Scene 6: 9
Scene 7: 10

波形：一路上升
症状：读者在 Scene 4 就累了
```

```
Scene 1: 8
Scene 2: 9
Scene 3: 10
Scene 4: 9
Scene 5: 8
Scene 6: 7
Scene 7: 6

波形：一路下降
症状：读者在 Scene 3 之后放弃
```

### 责编审节奏检查

责编在 Phase 4 必须对着 beat-sheet 画出情绪能级波形：

- 连续 2 场 ≥7 → 标红（审美疲劳）
- 连续 2 场 ≤3 → 标红（读者走神）
- 单集最高能级未达 ≥8 → 标黄（本集缺乏高光）

## 七、场景进入的四种开门方式

### 方式 1：对话开门

```
"你父亲的遗物里，有一封信。"
```

**优势**：信息密度最高，直接进入戏剧
**适用**：F2 推进 / F3 揭示 / F7 爆发
**风险**：缺乏氛围铺垫

### 方式 2：动作开门

```
林墨把烟盒在桌上按了三次。没点。
```

**优势**：人物状态立刻可见
**适用**：F1 建立 / F5 沉淀
**风险**：如果动作不够特别，读者没印象

### 方式 3：环境开门

```
凌晨三点的医院走廊。荧光灯管坏了两根，光是闪的。
```

**优势**：氛围感强
**适用**：F1 建立 / F6 过渡
**风险**：如果太长会让读者觉得"作者在炫技"

### 方式 4：第三方视角开门

```
保洁阿姨推着车从走廊经过时，看见 203 病房门口站着一个男人。
```

**优势**：新鲜感（不是主角视角）
**适用**：F3 揭示 / F4 反转（尤其制造信息差）
**风险**：视角切换成本高，一集不宜超过 1 次

### 单集开门方式配比

7 个场景推荐配比：

- 对话开门 × 3
- 动作开门 × 2
- 环境开门 × 1
- 第三方视角 × 1（可选）

**禁止全部同方式开门** = 单调。

## 八、场景退出的三种方式

### 方式 1：悬念切出

```
他的手机亮了一下。是一条短信。他看了一眼，把手机放下，没回复。
```

**读者感觉**："那条短信是什么？"
**适用**：F4 反转 / F8 收束

### 方式 2：动作戛然

```
他起身朝门口走去。走到一半，停下。他回头——
```

**读者感觉**："他看见了什么？"
**适用**：F3 揭示 / F4 反转

### 方式 3：情绪余韵

```
周伯关上门。书房里只剩他一个人。他走到窗前，把手表摘下来，重新戴好。
这一次是正着戴。
```

**读者感觉**：信息透露（周伯自己知道他之前戴反了）+ 情绪沉淀
**适用**：F5 沉淀 / F8 收束

### 禁止的退出方式

```
❌ 告别型退出：
  "那我走了。"
  "嗯，路上小心。"
  林墨走出茶馆。

❌ 总结型退出：
  这次谈话让林墨明白了很多事。他决定去查个明白。

❌ 下次预告型：
  明天还有更重要的事情在等着他。
```

## 九、场景设计的六个失败模式

### 失败 1：没有状态改变

**症状**：场景结束后，所有角色的关系/信息/情绪/决定都没变
**修法**：强制问"什么变了？"，如果答不出，删场景或重写

### 失败 2：功能不明

**症状**：这场戏既像推进又像揭示又像沉淀
**修法**：强制选**一个**主功能，其他功能作为副作用

### 失败 3：转折点缺失

**症状**：场景从头到尾一个节奏、一个情绪
**修法**：在 60-75% 位置强制加入一个转折（信息/情感/权力任一）

### 失败 4：信息差为 0

**症状**：读者、主角、其他角色知道的一样多
**修法**：强制配置至少一个信息差方向

### 失败 5：迟入早出未贯彻

**症状**：场景开头 200 字在"安排角色到位"
**修法**：删掉所有"到达/开门/坐下/打招呼"的过场

### 失败 6：情绪能级连续

**症状**：连续 3 场 7 分以上
**修法**：中间插一个 F5 沉淀场（能级 3-4）

## 十、场景学检查清单

编剧写完单个场景的 beat-sheet 后自查：

### 必答题
- [ ] 本场的主功能是什么？（F1-F8 选一）
- [ ] 进场时和离场时，什么变了？
- [ ] 转折点在哪个位置？是哪种类型？
- [ ] 主要信息差方向是什么？
- [ ] 情绪能级是多少？
- [ ] 与前一场的能级差是多少？

### 是非题
- [ ] 避免了"到达/开门/坐下"过场
- [ ] 不是和前一场同功能
- [ ] 不是和前一场能级相同
- [ ] 主角和对手反相位（若有对戏）
- [ ] 退出方式不是"告别/总结/预告"型

**任意一条未通过 → 重写该场景。**

## 十一、火影案例——鸣人与带土最终战的场景设计

拆解一个经典场景（疾风传带土回归战的"宇智波一族真相"场景）：

```yaml
scene_title: "第四次忍界大战 · 带土自述身世"
episode: "疾风传最终章"

# 功能标注
primary_function: "F3 揭示"
secondary_function: "F4 反转（带土的动机翻转读者认知）"

# 三幕结构（场景内部）
structure:
  setup (0-25%):
    content: "鸣人质问带土为什么背叛"
    emotion: "愤怒 + 困惑"
    
  development (25-70%):
    content: "带土开始讲琳的事——雨天、任务、雷切"
    emotion: "逐渐转向悲痛"
    
  turning_point (at 70%):
    content: "带土说'卡卡西杀了她' → 读者意识到这不是背叛，是创伤的反应"
    type: "信息转折 + 情感转折"
    
  climax (70-90%):
    content: "带土的面具第一次裂开"
    emotion: "爆发 + 释放"
    
  close (90-100%):
    content: "鸣人伸出手，但带土没接——两人重新对立"
    emotion: "余韵（和解失败的疼痛）"

# 信息差
information_asymmetry:
  primary: "读者 > 鸣人"
  detail: "读者早已知道带土是 Obito，鸣人不知道"
  tension: "读者一直等鸣人听懂 → 当他听懂时产生巨大情绪释放"
  
  secondary: "带土 > 读者"
  detail: "带土知道自己根本无法被鸣人感化（秘密）"
  tension: "读者一直希望和解，但带土的身体语言一直在告诉我们不可能"

# 反相位
phase_match:
```
