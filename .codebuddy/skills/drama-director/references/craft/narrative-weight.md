# 叙事重量 · Narrative Weight

> 本文件是 **编剧 / 责编 / 文学顾问** 的共用手册。
> 补全 prose.md（写好一句话）与 editing.md（评审）之间的缺口：**一场戏有没有重量**。
>
> 核心命题：**字数不是问题，叙事密度才是。**  
> 一场 1800 字的戏如果没有"什么变了"，它就是流水账；一段 300 字的闪回如果带来一个新事实，它比 2000 字的陈设更重。

---

## 一、什么是"叙事重量"

### 不等于字数

```
❌ 字数 ≥ 1500 = 这场戏有重量
❌ 描写多 = 这场戏有重量
❌ 对话多 = 这场戏有重量
```

### 等于"读者合上这一页时，世界是否变了"

```
✅ 叙事重量 = 读者在读这场戏之前和之后，对 (故事 / 角色 / 世界 / 自己) 的理解之差
```

**测试**：如果一个读者快速扫过这场戏，ta 漏掉的是"修辞"还是"事实"？
- 漏掉修辞 → 叙事重量充分（核心事实抓得住）
- 漏掉事实 → 叙事重量不足（被修辞稀释）
- 什么都没漏 → 这场戏本身信息量为零

---

## 二、场景重量三项测试

每场戏都必须能回答下面三个问题。**任一项答不出 → 这场戏没有必要存在，或需要重写。**

### 测试 1 · 不可撤销动作（Irreversible Action）

**定义**：有没有一个主角（或关键角色）做出的、在此场之后不能假装没做的动作？

```
✅ 示例：
- 林墨第一次主动开口问沈砚之一个问题（开过口的人，回不去沉默）
- 陈怀瑜打开抽屉让林墨看见药瓶（让外人看见过，就回不去"独自持有"）
- 李学工用指甲在帆布上刻"降"字（刻下去的字不会消失）

❌ 反例：
- 林墨看着窗外发呆（下一场他仍然可以在任何地方发呆）
- 沈砚之点了两个菜（下一场他也可以点两个菜）
- 林墨观察酒店地毯的花纹（观察不是动作）
```

**关键词**：观察不是动作、思考不是动作、感受不是动作。**只有"不可重置"的行为才是动作。**

### 测试 2 · 读者信息增量（New Info For Reader）

**定义**：读者在这场戏结束时，比进入前**多知道一个具体事实**？

```
✅ 示例：
- 沈砚之："右手。他只有喝酒的时候用左手。"
  读者增量：沈砚之对林父的日常有精细记忆
- 陈怀瑜电话："李学工眼球追一个方向，东偏北三十度。"
  读者增量：殷墟深处不止一个指向（方向分化）
- 陌生电话持续 8 秒呼吸，最后一口气比前面深
  读者增量：有人在跟踪林墨 · 跟踪者想说点什么但没说

❌ 反例：
- "酒店的地毯是红底棕花的。"（场景装饰 · 非故事事实）
- "林墨打开大衣内兜的药瓶晃了一下。"（重复已知事实）
- "沈砚之的鞋又干净了。"（如果没被主角/其他角色注意到 → 仅仅是氛围 · 不算信息）
```

**关键词**：装饰不是信息、氛围不是信息、复读不是信息。**必须是这场戏特有的、此前未知的事实。**

### 测试 3 · 状态变化（State Change）

**定义**：这场戏结束时，有没有一个"状态"从 X 变成 Y？

状态可以是：
- **关系**（两个角色之间的信任度 / 亲密度 / 权力差）
- **信息**（某人知道或不知道某件事）
- **物理**（某人物的物理位置 / 持有物 / 身体状态）
- **心理**（主角的防御姿态 / 锚点 / 阶段）

```
✅ 示例：
- Scene 3 晚餐：
  from: 林墨-沈砚之是"监护人 vs 被监护人"
  to:   林墨-沈砚之是"两个都听得见的人"（反相位的共犯）
  
- Scene 5 母亲电话：
  from: 母亲-林墨是"褪色开始（叫错名字）"
  to:   母亲-林墨是"不认识了（哪位）"——褪色第二阶段

❌ 反例：
- Scene X 林墨在房间里待了三十分钟什么也没发生
  （状态没变 = 这场戏可以删）
```

**关键词**：不是"情绪变了"算状态变化，**是"可以被另一个角色验证的变化"才算状态变化**。
- ❌ "林墨心情变沉重了"（心情变化只能主角自己知道）
- ✅ "林墨在笔记本封面里侧写了一个'二'"（任何人拿起笔记本都能验证）

### 三项测试的"两胜一负"规则

**单场最低标准**：三项中至少**两项**有明确的填写。

- 三项都有 → 场景重量充分
- 两项有 → 合格（通常是一场沉淀/过渡场：不可撤销动作可以少一些）
- 一项有 → 不合格，需要重写或合并
- 零项 → 立即删除这场

## 三、人物重量三项测试

同一场戏内，对**每个出场的有名字角色**都应该能回答：

### 测试 1 · 行动（Action）
该角色在这场戏中做了什么？——动作必须是可描述的物理动作，不是"在思考"。

### 测试 2 · 暴露（Exposure）
该角色在这场戏中暴露了什么？——可以是 SOUL 的一个字段、一个身体签名、一个以前读者不知道的事实。

### 测试 3 · 改变（Change）
该角色在离开这场戏时，是否与进入时有任何差异？——可以是信息层面（多知道一件事）或关系层面（对他人的 trust 变了）。

**合格标准**：三项中至少**两项**有填写。没有角色应该是"纯功能"的（除非明确标为 C 级背景）。

## 四、叙事时间的压缩与拉伸

### 为什么一段 500 字的闪回能顶 2000 字的陈设

**物理时间 ≠ 叙事时间。**

- 2000 字的陈设描写 → 叙事时间 = 0（什么也没发生）
- 500 字的闪回 → 叙事时间 = 10 年（人物在 10 年里变成了现在的样子）
- 1000 字的对话 → 叙事时间 = 5 分钟（但密度极高）

### 叙事时间的四种操作

| 操作 | 定义 | 典型字数/对应物理时间 | 用途 |
|---|---|---|---|
| **实时（Scene Time）** | 叙事时间 = 物理时间 | 1000 字/3-5 分钟 | 对话戏、动作戏 |
| **压缩（Summary）** | 叙事时间 >> 物理时间 | 300 字/半年 | 过渡、交代 |
| **闪回（Flashback）** | 叙事时间跳回过去 | 500 字/一段往事 | 动机揭示、身份暴露 |
| **拉伸（Stretching）** | 叙事时间 << 物理时间 | 500 字/2 秒 | 生理爆点、身体诗学、慢镜头 |

### 字数偏短时的正确处方

**错误**：机械补陈设（= 实时但空转）  
**正确**：检查是否缺了某种叙事时间操作

- 这一集有闪回吗？没有 → 能不能在一个关键节点插入 200-400 字闪回？
- 这一集有压缩吗？没有 → 能不能在过渡时用 100 字概述几天的变化？
- 高潮场有拉伸吗？没有 → 能不能把 2 秒动作拉成 300 字慢镜？

## 五、"呼吸段"与"流水账"的分界

### 呼吸段（Breathing Beat · 有意的留白）

**定义**：在高能场景前后故意留出的低能段落，给读者**消化情绪**的时间。

**特征**：
- 紧跟在一个爆点之后（F7 爆发 → F5 沉淀）
- 主角在做一件重复性、仪式性的小事（擦桌子、系鞋带、看窗外）
- 读者此时在"缓冲"，不是在"获取信息"
- 通常 ≤ 500 字

### 流水账（Water Writing · 失控的副作用）

**定义**：没有前置爆点、没有后置爆点，只是为了"写满字数"而存在的段落。

**特征**：
- 出现在任何位置（但最常在 F1/F6）
- 内容是陈设描写（墙纸 / 地毯 / 工牌 / 灯光 / 广告牌 / LOGO 印刷）
- 读者读完会问"这一段我错过了什么"
- 通常 100-300 字，但密集出现

### 区分原则

```
呼吸段：前面有爆点 → 读者需要缓冲 → 作者给呼吸 → 读者进入下一场更专注
流水账：前面没爆点 → 读者没理由缓冲 → 作者硬加描写 → 读者开始跳读
```

**自查**：一个段落如果从文件里删除，读者**会不会变得不懂故事**？
- 会 → 这不是流水账
- 不会，但"氛围变淡了" → 可能是呼吸段（保留），也可能是流水账（删掉）
- 不会，且"氛围也没变淡" → 肯定是流水账

## 六、反例合集 · AI 生成的"可见物补白"如何识别

### 反例 1 · 装饰品堆叠

```
❌ "酒店大堂不大。地毯是红底棕花那种，八十年代的纹样，但看得出是最近
    刚换的，边缘还没完全压平，翘起一点点毛边。空气里有一点点淡淡的木
    头味，大概是前台柜子本身的味道。柜子是深色的，立柱上贴了一圈金色
    的镶边。镶边有一小块脱了金漆，露出底下的灰色。"
```

**诊断**：
- 测试 1 不可撤销动作：无
- 测试 2 读者信息增量：无（地毯花纹、柜子材质、镶边剥漆均与后续无关）
- 测试 3 状态变化：无

**修法**：
- 删除 → 损失 0
- 或：把其中一项变成"有信息"的（例如柜子镶边剥漆处**藏了一张便利贴**，与故事有关）

### 反例 2 · 工牌/广告牌/LOGO 堆叠

```
❌ "前台是一个年轻女人，头发扎得很紧，脸上没什么表情。她胸口别了一
    个工牌，印着'前台·何琳'。林墨在她低头的时候扫了一眼这个名字。
    他没打算记住。"
```

**诊断**：
- 工牌上的名字"何琳"在后文从未出现
- "他没打算记住"—— 作者自己承认了这个细节的无用

**修法**：删掉整段，或让这个名字在后面出现（例如陌生电话其实是前台打的？这样名字就有重量了）

### 反例 3 · 车灯、路过动物、天气的堆叠

```
❌ "车灯的光从窗户上扫过，扫过餐桌，扫过两个人的手背，然后消失。"
```

**诊断**：
- 三连排比（扫过 A、扫过 B、扫过 C）—— 典型 AI 句式
- 读者增量：无（车灯扫过是日常）
- 作用声称是"呼吸段"—— 但前后无爆点，不成立

**修法**：
- 如果真需要呼吸，用**一个动作**代替（例如"林墨把筷子放下"）
- 如果场景需要"外部世界存在感"，用**一个可疑的外部事件**代替（例如"外面一辆车走得比该走的慢"）

### 反例 4 · "他知道……"开头的心理陈述堆叠

```
❌ "他知道沈砚之是在看后视镜。"
❌ "他知道里面还剩多少粒。"
❌ "他知道母亲的客厅是什么样子。"
```

**诊断**：
- "他知道 X"是作者替主角**总结**，不是主角**展示** ta 知道
- 触发 check-ai-taste.js 的 C5.1 规则

**修法**：
- 改为**行动**："林墨从副驾上看了三次后视镜，沈砚之也在看。"
- 改为**感官**："他不用数也知道药瓶还剩多少——他的手一拿起来重量就对。"
- 改为**直接陈述**（不用"他知道"开头）

### 反例 5 · 电梯/走廊/门牌号的枚举

```
❌ "506 在左边。507 的门把手上挂了一个'请勿打扰'的牌子。508 在最
    右边。走廊尽头是一扇安全门，门上的绿色指示灯是亮的。中间过道上
    方的顶灯有一盏是闪的，每隔大概十秒闪一下，别的几盏都正常。"
```

**诊断**：
- 空间设置对后续情节无影响（507 始终空着、顶灯闪一下与故事无关）
- 测试 2 读者信息增量：仅仅是"房间在 5 楼"这一件事，被稀释到 120 字里

**修法**：
- 浓缩到 20 字："506 和 508，中间隔着一间空房。"
- 或：让 507 在本集后面有戏（例如半夜从 507 传来声音）

## 七、诊断工作表（20 题 · 编剧+责编+文学顾问共用）

### A 场景重量（每场独立打分）

- [ ] 1. 本场有一个"不可撤销动作"吗？
- [ ] 2. 本场结束时，读者多知道一件具体事实吗？
- [ ] 3. 本场结束时，某个状态（关系/信息/物理/心理）变了吗？
- [ ] 4. 本场的转折点（60-75% 位置）能一句话说清是什么吗？
- [ ] 5. 如果删掉本场，后续情节还能接上吗？（接不上 = 本场必要）

### B 人物重量（对每个有名字角色）

- [ ] 6. 每个角色都在本场做了至少一个物理动作吗？
- [ ] 7. 每个角色都在本场暴露了至少一点自身吗？
- [ ] 8. 每个角色离场时与入场时有差异吗？

### C 叙事时间

- [ ] 9. 本集有至少一段"叙事时间压缩"（summary）吗？
- [ ] 10. 本集有至少一段"叙事时间拉伸"（慢镜）吗？（高潮场建议）
- [ ] 11. 本集是否需要一段"叙事时间跳跃"（闪回）？如需要，是否安排？

### D 语言与节奏

- [ ] 12. 呼吸段（F5 沉淀）前面是否有一个爆点作为锚？
- [ ] 13. 陈设描写段是否 < 全集 10%？
- [ ] 14. "他知道 X"类认知陈述段 ≤ 3 处？
- [ ] 15. 连续三个短独段 ≤ 5 处？

### E 读者密度

- [ ] 16. 本集至少有一个视觉锚（可以被截图分享）吗？
- [ ] 17. 本集至少有一句话可以被读者引用吗？
- [ ] 18. 本集至少有一处"主动引发读者推理"的地方吗？（信息差设计）

### F 综合

- [ ] 19. 本集 scene_weight 三项覆盖率 ≥ 80%？
- [ ] 20. 本集不存在"为了字数而加字"的段落？

### 评分标准

- ≥ 16/20：叙事重量达标（可放行）
- 12-15：部分缺陷，按缺的题目派活（具体见下表）
- < 12：结构有问题，回 Phase 2 重写 beat-sheet

### 题目 → 派活对照表

| 不过 | 派给 | 动作 |
|---|---|---|
| 1-5 | **编剧** | 改 beat-sheet，不是改正文 |
| 6-8 | **编剧 + 表演指导** | 调整角色 SOUL 激活 |
| 9-11 | **文学顾问** | 插入叙事时间操作 |
| 12-15 | **文学顾问** | 删/压缩流水账 |
| 16-18 | **编剧** | 改 beat-sheet 的视觉锚/金句/信息差 |
| 19-20 | **责编自己** | 出强制删改清单 |

## 八、反 "字数补白" 四大禁令

### 禁令 1 · 责编不得出"补到 XXXX 字"类 order

```
❌ 错误 order：
  "Scene 1 补白 +300 字到 1200"
  "本集字数总体 +1500 字"

✅ 正确 order：
  "Scene 1 的 irreversible_action 空缺，编剧需给出一个"
  "Scene 4 缺一段叙事时间压缩，文学顾问加 200 字 summary"
```

### 禁令 2 · 文学顾问不得以"陈设描写"回应字数不足

```
❌ 错误动作：
  补"地毯花纹 / 工牌 / 灯光 / 广告牌 / LOGO"

✅ 正确动作：
  补"闪回 / summary / 慢镜 / 身体签名 / 意象阶段推进"
```

### 禁令 3 · 字数短于门槛时，优先考虑**删场**而非**补场**

```
沉淀集 3800 字（差 200 到 4000）
  ❌ 补 200 字陈设
  ✅ 删 500 字过场 → 剩下 3300 → 承认"这是过渡集（下限 3000）"，position 改标

揭示集 6200 字（差 300 到 6500）
  ❌ 补 300 字陈设
  ✅ 先自查：本集是否真是揭示集？如果只是"推进集"标错了，改 position 到推进集，立即达标
```

### 禁令 4 · "position 声明"先于"字数判断"

```
流程：
  1. brief 决定 position（Phase 1 导演定）
  2. beat-sheet yaml 写 position（Phase 2 编剧延续）
  3. validate-beat-sheet 按 position 分级门槛判断
  4. 字数不足才进入叙事重量诊断

错误流程：
  1. 写完 novel 发现字数短
  2. 反推"那就是沉淀集吧"（事后调标签 = 作弊）
```

## 十一、诊断示例 · EP03 回溯

以 EP03 为例演示诊断：

### Scene 1 · 到安阳（900 字 → 实写 1300 字）

| 测试 | 结果 |
|---|---|
| 不可撤销动作 | **空**（林墨下车不是不可撤销的——他可以上车回去）|
| 读者信息增量 | 弱（"沈砚之今天绕路了"——但这件事没有后续）|
| 状态变化 | 弱（只是从车里到酒店大堂）|

**诊断**：Scene 1 本质是一场 F6 过渡，**建议字数压到 400-500**，而不是补到 900。
**修法**：责编应该给编剧派"合并 Scene 1 到 Scene 2 的开头，统一到 1500 字的 F1 建立场"。

### EP03 整集诊断结论

- 合格场：Scene 2, 3, 4, 5, 6
- 问题场：Scene 1（应压缩或合并）
- 现状总字数：4711
- 若 Scene 1 砍到 500 字：总字数 ≈ 4300
- position 应标：**沉淀集**（4000-6000）

**结论**：EP03 根本不需要补白。真正要做的是**把 Scene 1 砍一半**，而不是在每场戏里堆陈设。

---

> **叙事重量不是"字数" + "描写密度"的代名词。**  
> **它是"读者读完一场戏后，世界变了多少"的度量。**  
>
> 写作者最容易犯的错误：把场景当作**容器**，往里灌字数。  
> 好的场景是**事件**——读完之前和读完之后，是两个世界。
