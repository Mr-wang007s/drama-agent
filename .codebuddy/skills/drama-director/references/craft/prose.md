# 语言学 · Prose Craft

> 正文的诗意与节奏。
> 本文件是**文学顾问**的核心手册，**表演指导**审身体描写的参考，**责编**审文风的事实源。
> 核心命题：**好的正文不是"写得好看"，是"读得进去"**。读者不应该"欣赏"文字，读者应该"消失在"文字里。

---

## 一、正文写作的三级硬约束

本节迁移自 `.codebuddy/rules/writing-craft.md`。是正文产出的**机械硬门控**（由 `check-ai-taste.js` 自动检测）。

### 🔴 A 级硬约束（违反 = AI 味门控 Error，阻断）

**A1 · 破折号 `——` 全篇目标 ≤5**

- 硬上限 8（脚本上限），自律目标 ≤5，写作时默认不用。
- 想表达"进一步说明"→ 改句号另起
- 想表达"对偶"→ 改逗号陈述
- 仅允许在对话被打断情境中使用：`"我不是……"——他停下来，看我。`

**A2 · `**加粗**` 正文禁用**

novel.md 正文一律不用 Markdown 加粗。需要重音时，改用**句子结构**（短句独立成段）。

**A3 · `## 二级/三级标题` 正文禁用**

novel.md 用段落编号（中文一/二/三/四）或一个空行换场。目录标题（`# 第 X 章 · 标题`）允许一个，其余禁止。

**A4 · "不是 X，是 Y" 对偶句 ≤3 处/集**

该句式密集会形成节奏单调。优先扩展为**具象描述**：

```
❌ 不是冷，是麻
✅ 不是冷。是那种"我身体记得但我脑子不配合"的麻
```

**A5 · 前集连续性零伪造**

回指前集的**台词、场景细节、具体动作**必须能在前集 novel.md 里 grep 到原文。

写 beat-sheet 时要回指前集，必须先：
```bash
grep -n "关键词" stories/<story>/episodes/ep*/output/novel.md
```

beat-sheet 末尾**必填**"前集事实核对清单"块，列：回指内容 + 前集 ep-id + 原文行号。

**特例**：人物的**身体印象 / 感觉**（"林墨对这张侧脸有印象"）不受此约束——那是作者可自由扩写的角色内部经验。

**A6 · novel.md 严禁 EPxx 元标记泄漏**

集 ID 是工程元知识，不是角色的内心语言。林墨不会在心里想"EP04 那晚"——他想的是"昨晚"、"前天"、"工地夜那一天"、"三月十四号那晚"、"从病院出来那一晚"、"刚出山那几天"。

**常见错误替换表**：

| ❌ 正文禁用 | ✅ 角色语言 |
|---|---|
| EP01 那天 | 那天 / 从病院出来那天 / 三月十四号那天 |
| EP02 凌晨 | 工地夜那天凌晨 / 前天凌晨 |
| EP04 那晚 | 昨晚 / 回京那晚 |
| 章节标题 EP06 · 名 | 第六章 · 名 |

脚本 `check-ai-taste.js` 的 `episode_id_leak` 规则是 **error 级**，发现即阻断 Phase 5 门控。

---

### 🟡 B 级软约束（违反 = Critic/责编 Warning）

**B1 · 单集字数目标 6500-8500 中文字符**

- 硬下限 **6000 中文字符**
- 每场景在 beat-sheet 预算**字数 + 行数**双指标
- 每个 Beat 最少写 **3 层**：事件 + 角色身体反应 + 角色内心观察
- 每场景至少 **2 个慢镜头**：一件物品的细节 / 一段无事件的静默

**B2 · 单词独段 ≤5 处/集**

独占一行的短句（≤8 字、无句号）是强节奏符号，密度超 5 则成风格癖好。

**B3 · 三连排比 ≤2 处/集**

"A 的 B，C 的 D，E 的 F" 结构。

**B4 · 不做 Over-Connect（作者替读者接线）**

任何跨集伏笔的回声只铺**一次**，**不解释**。

```
❌ "陈教授说了一个'往'字。这是父亲梦里'往下还有一层'的前半截。"
✅ "陈教授说了一个'往'字。林墨点头，他不让自己把下半截在脑子里接出来。"
```

用角色的**身体反应**代替认知串联。

**B5 · 视角不越界**

- 主视角人物不能使用他 `SOUL.known_facts + MEMORY.md` 之外的信息
- S 级角色的 `secret` 字段在卷内合约期内**绝对不泄露**给读者

---

### 🟢 C 级正向约束（违反 = 责编扣分）

C 级约束是"写得好"的正面要求——不是"别犯错"，而是"必须做到"。

**C1 · "读者替接话"≥1 处/集**

至少一处对话/动作让读者脑中自动冒出下半句（但正文不写出来）。

```
❌ "他知道这不对。"
✅ "他闭嘴了。他知道自己下一句会说什么，所以他闭嘴了。"
```

**C2 · "沉默比发言主动"≥2 处/集**

至少两处角色**选择不说**比说更有力的瞬间。

**C3 · 钩子回收权重 ≥2/集**

每集必须回收至少 2 个前集埋下的 A/B 级钩子。A 级钩子回收权重 ×2，B 级 ×1。

**C4 · 角色语言 Jaccard < 0.25**

任意两个 S 级角色的对话高频词相似度 < 0.25。由责编定性判断（详见 `dialogue.md`）。

**C5 · 句式黑名单 10 条**

| 编号 | 模式 | 级别 | 上限 |
|---|---|---|---|
| C5.1 | "他/她知道……" 开头段 | warning | 3/集 |
| C5.2 | "这一刻，……" | warning | 2/集 |
| C5.3 | "仿佛/好像/就像……一样" 明喻堆叠 | warning | 4/集 |
| C5.4 | "某种（说不清的/莫名的）……" | error | 2/集 |
| C5.5 | "他不知道为什么……" | warning | 2/集 |
| C5.6 | "心里（涌起/升起/浮起）一股……" | error | 1/集 |
| C5.7 | "时间（仿佛/好像）（停止/静止）了" | error | 0/集 |
| C5.8 | "（深深地/重重地）吸了一口气" | warning | 2/集 |
| C5.9 | "眼眶（微微/不由自主地）泛红" | error | 1/集 |
| C5.10 | "嘴角（微微/不自觉地）上扬" | error | 1/集 |

这些句式是 AI 写作的高频坏习惯。黑名单由 `check-ai-taste.js` 自动检测。

## 二、破防戏五铁律（R1-R5）

当 beat-sheet 标记某场景为"破防戏/breakdown"时，该场景必须满足以下全部 5 条。

### R1 · 前 200 字过度平静

破防前的铺垫必须是日常/过度正常的——越平越好。

角色在做一件无关紧要的小事（叠衣服/数硬币/看菜单）。

**读者在这 200 字里不应该预感到崩溃即将到来。**

### R2 · 触发物 ≤5 字

真正的引爆点必须极短——一个词/一句话/一个声音/一个气味。

```
❌ "他看着父亲的照片，想起了那些年父亲对他的期望与失望……" (30字)
✅ "照片里的手。" (5字)
```

### R3 · 第一个动作不是情绪动作

崩溃后角色的**第一个可观察行为**不能是哭/怒/吼——必须是一个不合时宜的日常动作。

**样板**：
- 继续削苹果但削进了手指
- 把杯子放下但放偏了桌沿
- 继续走但走错了方向

**原理**：读者通过"不对劲的正常"感知到崩溃，比直写情绪有力 10 倍。

### R4 · 无关者目击

破防发生时必须有一个"不知情的在场者"视角切入（≥1 句）。

这个人不理解发生了什么，他的困惑/尴尬/无动于衷反衬当事人的痛苦。

```
❌ 独角戏纯内心崩溃
✅ 旁边卖豆浆的老头看了他一眼然后继续磨豆子。
```

### R5 · 清醒 ≥ 2× 崩溃

崩溃段落字数 ≤ 收尾清醒段落字数的 1/2。

即：写崩溃用 150 字，写"回来"至少用 300 字。

**原理**：快崩慢收 = 真实；慢崩快收 = 廉价煽情。

### 完整破防戏样本（林墨 EP05）

```
（R1: 前 200 字过度平静）

林墨回到出租屋已经是半夜。他脱了外套，挂在门后。他从冰箱里拿出
一盒昨天剩的饺子——十五个，数了一下，十五个。他把饺子放进微波
炉，按了两分钟。他站在厨房里等。微波炉的灯是暖黄色的，把他的
影子投在墙上。他的手放在台面上，指节上有一道新结的痂。他看了
一眼那道痂，没动。

微波炉响了。他把饺子端出来，放到餐桌上。他在桌前坐下，拿起筷子。

（R2: 触发物 ≤5 字）

"十四个。"他说。

（R3: 第一个动作不是情绪动作）

他把筷子放下。他走到厨房，打开微波炉——里面是空的。他打开冰箱，
从上到下看了一遍。他打开垃圾桶。他蹲下来，看桌子底下。他站起来，
又回到餐桌前，数了一遍：十四个。

他打开冰箱，重新拿出一盒没动过的饺子。他把它放进微波炉，按了两
分钟。他等。

（R4: 无关者目击）

楼下的保安正好在夜巡，经过他的窗口，看见这个男人站在厨房里盯着
微波炉。保安以为这人在发呆，摇摇头继续走。

（R5: 清醒 ≥ 2× 崩溃）

微波炉又响了。林墨这次没动。他在餐桌前坐了很久。他不记得自己
坐了多久。后来他站起来，把两盒饺子都倒进了垃圾桶。他把垃圾袋
扎紧，提到门外。他回来洗手，洗了很仔细的两遍。他擦干手，关了
厨房的灯。

他走到卧室，没开灯。他坐在床沿，脱鞋。左脚的袜子他脱了两次——
第一次忘了还没脱鞋。他终于脱下鞋和袜子，把脚放进被子里。被子是
凉的。

他躺下。他睁着眼睛看着天花板，看了有半个小时。他没哭。他没想什么
具体的事。他只是躺在那里，等天亮。

后来他睡着了。

第二天早上他醒来，像什么都没发生一样。他洗脸，刷牙，泡面吃。他
在镜子里看了自己一眼——眼下有一点点青。他没在意。他收拾好东西，
出门去见周伯。他甚至哼了一小段《东方红》——他父亲以前经常在
早上哼的那个调子。

他没意识到自己在哼。
```

### 破防戏的常见失败

**失败 1：触发物太长**
```
❌ "他看着父亲的旧相册，翻到 1987 年的夏天，那年父亲刚三十岁，
     带他去了老家的河边，父亲教他游泳，他呛了一口水——"
→ 30 字以上的铺垫已经破坏了 R2
```

**失败 2：第一反应是哭**
```
❌ "眼泪不受控制地流下来。"
→ 违反 R3。第一反应必须是不合时宜的日常动作。
```

**失败 3：独角戏**
```
❌ 全程只有主角，没有第三方视角
→ 违反 R4。必须有"不知情的在场者"。
```

**失败 4：快崩快收**
```
❌ 崩溃写 500 字 + 收尾 100 字
→ 违反 R5。正确比例是崩溃 150 字 + 收尾 300+ 字。
```

## 三、意象系统——反复出现的诗意回响

### 什么是意象

**意象 = 在故事中反复出现的"感官锚点"**。

它可以是：
- 物品（父亲的怀表、母亲的围巾）
- 环境（雨、月光、金属门）
- 感官（某种味道、某种声音）
- 动作（擦眼镜、点烟、转杯子）

关键是：**反复出现 + 每次出现的含义在微妙变化**。

### 意象系统的三阶段

Umberto Eco 在《符号学与语言哲学》中论述：一个意象在故事中必经三阶段：

**阶段 1：引入（Introduce）**
- 首次出现
- 建立感官锚点
- 含义是初始的、中性的

**阶段 2：挑战（Challenge）**
- 再次出现但被颠覆/变形/对立使用
- 含义开始偏离初始
- 读者开始注意到"这个东西反复出现"

**阶段 3：完成（Complete）**
- 在新语境中重生
- 获得系列性含义
- 读者回望时感到"啊，这个东西一直在"

### 样例：林墨故事中的"手表"意象

```
EP01：父亲遗物里有一块手表（引入·阶段 1）
  含义：父亲的象征、遗物

EP02：周伯的手表戴反了（挑战·阶段 2）
  含义：手表从"遗物"变成了"线索"——周伯为什么要戴反？

EP04：林墨在梦里看见父亲摘下手表放在桌上（挑战·阶段 2 延续）
  含义：手表与父亲的关系深化——他不仅留下了它，也曾摘下过它

EP07：真相揭晓时，父亲的手表里藏着一张小字条（完成·阶段 3）
  含义：手表是密码，是父亲留下的最后遗言
```

**这就是意象系统**。从"一个物品"演变为"一个主题"。

### 核心意象的数量

每个故事推荐 6-10 个核心意象。每集激活 2-3 个（不能全部同时在场）。

### 意象台账

每个故事在 `world/imagery-ledger.md` 维护（由编剧+文学顾问在 Phase 2 和 Phase 4 共同维护，**不再用脚本自动化**）：

```markdown
# Imagery Ledger · stories/<name>

## 核心意象

### 意象 1：父亲的手表
- 阶段：阶段 2（挑战）
- 引入：EP01 L234
- 历次出现：EP01 / EP02 / EP04
- 当前含义：从"遗物"演变为"线索"
- 计划到阶段 3：EP07（揭晓字条）

### 意象 2：金属门
- 阶段：阶段 1（引入）
- 引入：EP01 L56
- 当前含义：病院 / 父亲死的地方 / 创伤锚点
- 待挑战：未安排

### 意象 3：茉莉花茶
...
```

### 意象的误用

**误用 1：意象密度过高**
- 同一场戏激活 5+ 个意象 → 读者眼花
- 修法：每场戏聚焦 1-2 个

**误用 2：意象无演变**
- 一个意象出现 5 次，每次都是同样含义 → 不是意象，是重复
- 修法：每次出现必须有"微妙变化"

**误用 3：意象过度解释**
- "手表象征着父亲的缺席"——作者在讲解符号学
- 修法：意象的含义只能通过**读者自己感受**获得，不能被作者说出

## 四、身体诗学——让身体替心理说话

### 核心原则

**不写心理活动，写身体动作。**

这个在 `characterology.md` 已详细论述。本节补充**文本层面**的实践。

### 身体描写的三个层次

**层次 1：动作（Action）**

```
他点了烟。
```

层次 1 是事件。信息密度低。

**层次 2：动作 + 细节（Action + Detail）**

```
他点了烟。第一口吸得很深，第二口就只是夹着。
```

层次 2 有状态。通过动作的"异常"传递情绪。

**层次 3：动作 + 细节 + 环境互动（Action + Detail + Environment）**

```
他点了烟。第一口吸得很深，烟灰在烟头上凝住没落下来。他把烟夹在
手指间，让它自己烧。窗外的风把烟雾吹偏，但烟灰仍然没落。
```

层次 3 有诗意。身体、物品、环境共同构成一个"心理状态的客观对应物"（Eliot 语）。

### 单场戏的身体描写配比

推荐：

- 层次 1 的身体描写：40-50%（保持叙事推进）
- 层次 2 的身体描写：35-45%（塑造角色状态）
- 层次 3 的身体描写：10-15%（制造诗意锚点）

过高的层次 3 比例 = 全文"装腔作势"
过低的层次 3 比例 = 全文"流水账"

### 身体描写的节奏

```
❌ 连续多段都是层次 3：
  林墨点了烟，第一口吸得很深，烟灰在烟头上凝住。他把烟夹在手指间。
  他的呼吸很轻，像刻意压着。烟雾从他的鼻孔里冒出来，在他的眉毛上
  绕了一下才散。他的左手摸了摸桌沿，桌沿上有一道裂痕。
  
→ 每句都在慢镜头，读者疲劳。

✅ 层次混合：
  林墨点了烟。第一口吸得很深（层 2），第二口就只是夹着（层 2）。
  
  周伯看着他（层 1）。"你想问什么？"（对话）
  
  "没什么。"林墨把烟放在烟灰缸边（层 1），让它自己烧（层 3：细节+环境）。
  
→ 大部分是层 1-2，关键瞬间才进入层 3。
```

## 五、留白艺术——读者自己接话

### 什么是留白

**留白 = 作者该说的没说，让读者自己在脑中补完。**

中文古典文学的最高美学。

### 留白的三种实现

**实现 1：对话的留白**

```
❌ 
"你恨我吗？"
"我不恨你，只是觉得你做错了。"

✅ 
"你恨我吗？"
（林墨没答。他喝了一口茶。）
"我先走了。"
```

**实现 2：情节的留白**

```
❌
林墨打开了父亲的遗书。信里写着父亲的真实遭遇——他被方士会绑架，
被迫做了一件违心的事，他一生都在后悔……

✅
林墨打开了父亲的遗书。
他看到第三行停下了。
他把信纸合上，放回信封。他再也没打开过。
```

**实现 3：情绪的留白**

```
❌
林墨站在窗前，看着外面的夜色。他感到深深的悲伤，这种悲伤来自
他对父亲的思念，也来自他对自己的无力感……

✅
林墨站在窗前。楼下的超市关门了。
他不记得自己站了多久。
```

### 留白的节奏感

**一个场景留一处**是标准。

太多留白 = 读者负担（什么都要自己脑补）
太少留白 = 读者喘不过气（什么都告诉你）

### 信任读者

留白的本质是**信任读者**。

作者不相信读者能感受到父亲的遗书有多重 → 写一段"深深的悲伤"
作者相信读者能感受到 → "他看到第三行停下了。"

**信任读者 = 尊重读者 = 读者回以忠诚**。

## 六、反 Over-Connect——别替读者接线

### 什么是 Over-Connect

**作者把本应由读者自己完成的联想/推理/情感连接，写在了正文里。**

### 样例

```
场景 1：陈教授对林墨说："往下还有一层。"
  
场景 2（第二集）：林墨做了一个梦，梦见父亲说："往下还有一层。"

❌ Over-Connect 版本（作者接线）：
  林墨猛然想起——陈教授说过一模一样的话！这不是巧合。父亲和
  陈教授之间一定有什么关联。是不是父亲死前见过陈教授？

✅ 无 Over-Connect 版本（让读者接线）：
  林墨醒来时，这句话还在他耳朵里转。
  他坐起来，下床，走到厨房倒了一杯水。
  水太凉。
```

后者：读者自己会把两次"往下还有一层"连起来。**这个连接的瞬间就是读者的阅读快感。**

### Over-Connect 的五种常见形式

**形式 1：显式串联**
```
❌ "这和之前发生的事一模一样。"
```

**形式 2：解释隐喻**
```
❌ "那块手表象征着父亲的缺席。"
```

**形式 3：串联情绪**
```
❌ "林墨的心里涌起一股复杂的情感——有恨，有爱，有遗憾，有愧疚。"
```

**形式 4：回顾前情**
```
❌ "他想起了三年前父亲去世那天，他当时在病院里。EP01 他从病院出来时的情景还历历在目。"
```

**形式 5：预告后情**
```
❌ "他不知道，这个决定将改变他的一生。"
```

### 检验方法

**拿出笔在正文里圈一下所有"串联/解释/分析"的句子。如果多于 5 句/集，那是 Over-Connect。**

### 文学顾问的职责

文学顾问在 Phase 4 润色时，**专门抓 Over-Connect**。

步骤：
1. 通读全文，圈出所有"串联 / 解释 / 分析 / 回顾 / 预告"句
2. 对每一句问：读者能不能自己完成这个连接？
3. 如果能 → 删掉或改为"身体反应替代"
4. 如果不能 → 检查前文是否铺垫不足，补铺垫而不是解释

## 七、长短句节奏

### 长短句的定义

- **短句**：≤10 字
- **中句**：10-25 字
- **长句**：≥25 字

### 节奏配比

推荐 1:2 或 1:3（长:短）的比例。

```
1:2 比例（紧张型叙事）：
  短短长 / 短长短 / 长短短

1:3 比例（抒情型叙事）：
  短短短长 / 长短短短 / 短长短短
```

### 节奏的"炸点"功能

**炸点 = 信息/情感/转折的释放瞬间**。

炸点通常放在**节奏的异常位**：

```
节奏 1：长长短（长句铺垫，短句炸点）
  "这三年来他一直以为自己知道父亲最后想说什么，也一直以为自己
  可以等到真相揭晓那一天。他错了。"
  
节奏 2：短短长（短句积压，长句释放）
  "他愣住。他坐下。他没再说一句话——因为他意识到周伯刚才那句话
  不是安慰，是警告。"
```

### 节奏失败的症状

**症状 1：全短句**
```
他站起来。他走到门口。他停下。他回头。他看见她。他没说话。他离开。
```

→ 像 AI 小说。读者心跳会乱。

**症状 2：全长句**
```
他站起来走到门口，在门口停了一下，回头看了她一眼，但他没有说话，
然后他离开了，没有再看第二眼，也没有解释为什么。
```

→ 像作文。读者喘不过气。

**症状 3：节奏无炸点**
```
他站起来。他走到门口停下来回头看了她一眼。他没说话就离开了。
```

→ 节奏不错但没有信息释放点，读者感觉"无事发生"。

### 自检方法

随机抽一段 200 字，标记每句的字数：

```
"你来了。"（4）
周伯已经等了四十分钟。（11）
林墨在他对面坐下。（9）
茉莉花茶是周伯替他点的——三年前林墨父亲的忌日，周伯也给他点过这个茶。（39）
林墨看着茶杯，没抬头。（11）
"你父亲的遗物里，有一封信。"（13）
```

节奏：4-11-9-39-11-13

比例：短-中-中-长-中-中 → 1 个长句炸点 + 其他短中句，合格。

## 八、叙事视角的"呼吸感"

### 三种视角

**全知视角（Omniscient）**
- 作者看到所有角色的内心
- 适合史诗、复杂群像

**限知视角（Limited）**
- 跟着一个角色的视角
- 最常见、最实用

**多焦点视角（Multiple Limited）**
- 不同章节跟着不同角色
- 适合悬疑、复杂结构

### 视角的呼吸

即使在限知视角内，也要有"视角的呼吸"——偶尔拉远，偶尔推近。

**推近样本**：

```
林墨的左手在颤抖。他不确定是因为冷还是别的什么。他把左手藏进
袖子里。袖口的线头磨着他的手腕——那是昨天新买的衬衫，领标还在。
```

→ 视角紧贴林墨的身体感受。

**拉远样本**：

```
凌晨三点的老城区没什么人。路灯是黄色的，两盏之间有一段是黑的。
林墨从巷子的这头走到那头，影子在他脚下转了一圈。
```

→ 视角拉到整个空间，林墨成为空间中的一个元素。

### 推拉的节奏

单集推荐：每 500-800 字有一次视角推拉。

太多推 = 窒息感
太多拉 = 疏离感
完美比例：80% 贴近 + 20% 拉远（点缀式）

## 九、文学顾问的工作流

### Phase 4 润色阶段（责编诊断后介入）

文学顾问在责编出修订指令后介入，职责包括：

**职责 1：反 Over-Connect**
- 通读 novel.md，圈出所有"串联/解释/分析"句
- 改写为"身体反应 + 读者自己接线"模式
- 输出：Over-Connect 清理清单

**职责 2：长短句节奏优化**
- 对节奏失败的段落做微调
- 重点是"炸点位置"的检查
- 输出：节奏修订建议

**职责 3：意象系统维护**
- 更新 `world/imagery-ledger.md`
- 检查本集激活的意象是否在正确的阶段
- 提醒未来集的计划演进

**职责 4：破防戏合规（若有）**
- 对照 R1-R5 逐条检查
- 不合规处给出改写示范

**职责 5：身体描写层次检查**
- 统计层 1/2/3 比例
- 对"慢镜头过多"或"流水账"的段落给出改写

### 润色的边界

**文学顾问可以改**：
- 修辞细节
- 长短句节奏
- 身体描写层次
- Over-Connect 清理

**文学顾问不能改**：
- 情节
- 角色行为
- 对话的核心含义
- 场景功能

涉及后四项 → 回到编剧/责编处理。

## 十一、常见陷阱

### 陷阱 1：破折号惯性

```
❌ 习惯性用破折号做"进一步说明"：
  他看着她——那双眼睛他认识。——三年前他见过。
  
✅ 改用句号 / 逗号：
  他看着她。那双眼睛他认识。三年前他见过。
```

### 陷阱 2：对偶句滥用

```
❌ "不是冷，是麻。不是怕，是累。不是恨，是……"
→ 密集对偶 = 朗诵腔
  
✅ 具象化扩张：
  不是冷。是那种"我身体记得但我脑子不配合"的麻。
```

### 陷阱 3：比喻来源域混乱

```
❌ "他的心像一朵枯萎的花，像一片被风吹走的叶子，像暴雨中的船。"
→ 一个情绪用三个来源域不同的比喻 = 作者炫技
  
✅ 来源域统一：
  他的心像一艘没有锚的船。水在哪他就往哪飘。
```

### 陷阱 4：意象过度

```
❌ 一场戏激活 5 个意象（手表+茉莉花+金属门+父亲的笔+母亲的围巾）
→ 读者抓不住重点
  
✅ 一场戏激活 1-2 个核心意象，其他作为背景元素
```

### 陷阱 5：视角漂移

```
❌ 同一段里视角从林墨跳到周伯再跳回林墨
→ 读者晕。
  
✅ 限知视角坚持到底。视角切换至少隔一个空行。
```

### 陷阱 6：描写过度具体化

```
❌ 林墨 1.73 米，穿着深蓝色条纹衬衫，衬衫是 2022 年在三里屯优衣库
   买的，标签还没剪。他脚上穿的是黑色真皮牛津鞋……
→ 信息过载，没有状态传递
  
✅ 林墨的衬衫领标还没剪。
→ 一个细节传递 1 个信息（新买的/不熟悉的）
```
