# 回合制交互手艺（interaction-craft）

> 这是 Director 驱动 Team 的"施压 + 对抗"能力册。
> 回答的是："我 spawn 了 world-manager 和几个 Agent 之后，接下来如何让他们真正演出一场戏，而不是各自独白？"
>
> 先读 team-protocol.md 获取流程约束，再读本文获取手艺。

---

## 失败模式档案（读一遍再动手）

> 这些失败来自真实流水线，不是假设。下面每种写法都在 EP01 v2 重写中发生过，被用户一眼看穿。

### 失败 1 · "独白拼图"

**症状**：spawn 6 个 Agent，每个 Agent 输出 500-900 字第一人称完整场景。Director 把 6 份独白拼接成 novel.md。

**读者体验**：像读 6 个不同人的日记，不是看戏。

**根因**：Agent prompt 要求"输出你的视角场景"——这让每个 Agent 进入**"我来讲一遍"**心态，而不是**"我来演一拍"**心态。

### 失败 2 · "并行自述"

**症状**：多个 Agent 被 spawn 但互相不知道对方在说什么，各自在自己的时间线里演完了全场。

**读者体验**：同一件事被 4 个人讲 4 遍，张力为零。

**根因**：Team 协议被理解为"并行生成"，而不是"串行交互"。

### 失败 3 · "客套循环"

**症状**：两人对话"你说得对""我理解""你说呢""嗯"……来回五六轮没有推进。

**根因**：Agent 的 prompt 里没给"私人议程"——他不知道自己来这个场景**想要什么**，只能做表面客套。

### 失败 4 · "全员内心暗涌"

**症状**：每个 Agent 每回合都输出大段 [inner_thought]，场景里全是心理活动没有物理事件。

**根因**：Agent prompt 过度强调"内心"而忽视"行动"。

### 失败 5 · "world-manager 当旁白"

**症状**：world-manager 只描述环境（"风很大""窗外阴天"），不施压。角色被动漂浮。

**根因**：world-manager 被定位为"环境描写官"而不是"导演代理人"。

## world-manager 的 5 种施压策略

world-manager 不是旁白——它是导演代理人。每 3 回合必须注入一次有效施压。五种类型：

### 1 · 物理施压（最常用）

**定义**：通过**物品状态变化 / 光线 / 声音 / 温度**改变场景张力。

**示例**：
- 紫砂壶的壶盖滑了一下
- 窗外灰喜鹊飞走
- 电话响了半声又停
- 茶凉了
- 日光灯闪了两下
- 保温饭盒放到桌上的声音

**使用时机**：当对话进入"客套循环"时，注入物理变化强制角色反应。

**注意**：物理施压不是环境描写。描写 = "风很大"；施压 = "风把她围巾吹起来盖住她半边脸，她第一次露出她脖子上那道疤"。

### 2 · 信息施压

**定义**：让场景里某个**物品被注意到**（不一定被动 = 被主动放置）。

**示例**：
- 周伯把红塔山空烟盒放到桌上（林墨看见）
- 李医生把私人电话卡片推过去
- 有人放下一张照片

**使用时机**：当你需要让一个没说破的线索进入场景时，不要让角色说出来——让物品说出来。

### 3 · 第三方施压

**定义**：让**不出声的第三个人**出现在场景里，用他的目光/反应改变两人的对话权力。

**示例**：
- 张教授坐在沙发一角（周伯 + 林墨 + 张教授三人戏）
- 前台护士拿笔问"林妈妈？"（林墨突然要决定写什么名字）
- 走廊里有脚步声经过但不停

**使用时机**：两人对话陷入私密情感时，引入第三方强制双方回到"公开场合"的语域。

### 4 · 时间施压

**定义**：用**时间节点**制造不得不的选择。

**示例**：
- 保温饭盒还在散热（她已经到了四十分钟还没走）
- 车开上匝道（错过此出口下一个口三十公里）
- 沈砚之比约定时间早到十二分钟
- 窗外晨光切到桌沿（光在移动，时间在走）

**使用时机**：角色在犹豫不决时，让时间物理化显现。

### 5 · 身体施压

**定义**：让**角色的身体出现反应**（非自愿）——不是他想做这个动作，是他的身体替他做了。

**示例**：
- 林墨的食指在桌下敲到第四下停（他自己没察觉）
- 他的脖子僵了一下
- 他的鼻子酸了一下

**使用时机**：主角的理性在压抑情绪时，让他的身体背叛他。

## 回合调度序列（world-manager 的调度权手册）

### 对手戏（2 人）调度

```
回合 0  · world-manager broadcast 开场（环境 + 时间 + 在场） ≤ 200 字
回合 1  · @主场者：施压（物理施压 / 问句）
回合 2  · @客场者：反应（身体反应或沉默或短对白）
回合 3  · @主场者：加压（追问或物品呈现）
回合 4  · @客场者：转移（自嘲 / 无关话题 / 沉默）
回合 5  · world-manager 注入物理事件（如鸟飞走 / 茶凉）
回合 6  · @主场者：用新事件加压
回合 7  · @客场者：被迫直面（这是"戏眼"）
回合 8  · @主场者：退或进（给客场者留或夺）
回合 9  · @客场者：做决定或身体反应
回合 10 · world-manager broadcast 收场（物理节点：签字/起身/门关）
```

总轮次 10-14 次合适，≤ 20 强制结束。

### 群戏（3+ 人）调度

```
回合 0  · world-manager broadcast 开场（标明每人的空间位置）
回合 1  · @主场者：起始动作（泡茶 / 打开笔记本）
回合 2  · @主角（客场）：观察主场者的动作
回合 3  · @旁观者：一个微动作（点头/调整坐姿）
回合 4  · @主场者：引入核心议题
回合 5  · @主角：反应
回合 6  · @旁观者：看了一眼主角的手/脸（目击关键身体信号）
回合 7  · @主场者：加压
回合 8  · @主角：决定
回合 9  · @旁观者：起身 / 说第一句话（旁观者破壁是大招）
回合 10 · world-manager 收场
```

**旁观者破壁**是群戏的高级手法——全场都不说话的人最后说一句，权重是主角的三倍。

### 独幕戏（1 人）调度

```
回合 0  · world-manager broadcast 环境（角色独处的空间）
回合 1  · @主角：起始动作（醒来 / 坐下 / 看见）
回合 2  · world-manager 物理事件（耳钉响 / 桌上多了一张纸）
回合 3  · @主角：观察 + 动作（不是大段内心）
回合 4  · world-manager 物理事件（老王走过不抬头）
回合 5  · @主角：动作 + 极短内心闪念
回合 6  · world-manager 物理事件（光的变化）
回合 7  · @主角：决定性动作（离开空间 / 写下三个字 / 拿起物品）
回合 8  · world-manager 收场
```

独幕戏 world-manager 每 2 回合必须施压，因为没有对手。

## Agent 回合的三层结构

每个 Agent 每回合 ≤ 150 字。怎么用这 150 字？

### 第 1 层 · 对上个回合的身体反应（先）

**先反应，再说话**。Agent 被 @ 后的第一动作不是开口，是身体对上一个事件/对白的反应。

**示例**：
- 上个回合对手说了一句扎心的话。这个回合 Agent 第一动作：他愣了半秒。
- 上个回合 world-manager 让耳钉响了。这个回合 Agent 第一动作：他的手停在茶杯边。

### 第 2 层 · 动作 或 对白（主）

反应之后才是行动。行动有三选一：

- **动作**（推动场景）：他把两只手放到膝盖上。他合上病历。他站起来。
- **对白**（推动对话）：≤ 2 句短对白。
- **沉默**（最高级）：他没接话。他只是看着对方的笔。

### 第 3 层 · 极短内心闪念（可选）

**每 3 回合 ≤ 1 次**。内心闪念 ≤ 20 字，用自由间接引语：
- "他想：三年我没听过他这么锋利的话。"
- "没关系，他想。"
- "没说。"

**绝对禁止**：大段内心独白独占一个回合（那是独白不是 beat）。

## 两种致命的施压误区

### 误区 1 · "告诉 Agent 该怎么反应"

❌ world-manager 的 prompt 写："你告诉林墨他应该发现红塔山烟盒是父亲的。"

✅ world-manager 的正确做法：把物品呈现出来，让 Agent 自己决定他看见什么、怎么反应。

### 误区 2 · "替 Agent 做决定"

❌ world-manager 说："林墨决定签字。"

✅ world-manager 正确做法：推来聘书，放下钢笔——Agent 自己决定签或不签。

## 交互结束判定

world-manager 看到以下任一信号即结束场景：

- **beat-sheet 触发点全部覆盖**（最正规）
- **主角做出了那个决定**（签字 / "我去" / 起身）
- **连续 3 回合无新信息**（原地打转，强制结束）
- **物理空间变化**（门关上、车发动、光线由明转暗）
- **回合数 ≥ 20**（硬上限防死循环）

结束时 world-manager 必须 broadcast 一段**场景余韵**（≤ 100 字环境描写）作为场景尾音。

## 实战：一个合法的 5 回合交互片段

背景：林墨 × 李医生，面谈室 08:20，李医生准备给林墨私人电话卡片。

```
【world-manager broadcast】
面谈室在二层东南角。窗玻璃没开。八点二十。对面那把椅子靠背第四根竖木有一道细裂痕。

【world-manager @ 李医生】
你已经坐了六分钟。他推门进来。你左手按着病历右上角。你的任务：在他起身之前把卡片给他。

【李医生 reply】
李医生的左手还按在病历上。他数到六。"你昨晚睡得怎么样。"

【world-manager @ 林墨】
李医生问了你昨晚睡得怎么样。你坐下了。

【林墨 reply】
他坐下。两只手放在膝盖上。"跟前天一样。前天跟大前天一样。这就是精神病院的好处。"他自己说完停半秒。他没笑。

【world-manager @ 李医生】
他说了一句自嘲。他的食指在桌下敲到第四下停。你看见了。该你。

【李医生 reply】
李医生没接话。他看着林墨。他把左手从病历上挪开。他合上病历。"林医生。"他停了。"林墨。"

【world-manager @ 林墨】
他叫了你"林墨"。他合上了病历。

【林墨 reply】
他抬眼。
```

这 5 回合已经演出了 EP01 第 2 场戏的**前 20%**。后面的推-拉-转-收继续按同样节奏走。

注意：
- 每个 Agent 回合 ≤ 80 字
- 每回合都有**身体动作**（合病历、按病历、抬眼）
- 对白极短（1-2 句）
- world-manager 不描写环境，**施压**——"该你""你的任务"

## 给 Director 的自检

启动 Team 之前问自己：

- [ ] 我 spawn 的 world-manager prompt 里有没有"你是导演代理人，每 3 回合必须施压"这句话？
- [ ] 我 spawn 的每个 Agent prompt 里有没有"每回合 ≤ 150 字 ≤ 1 个 beat"这个硬约束？
- [ ] 每个 Agent 的"私人议程"是不是清晰？（他来这个场景想要什么）
- [ ] 主场者和客场者是不是明确？（先开口权）
- [ ] 我是不是已经禁止了"输出完整场景自述"？

五问全过 → 启动 Team → 观察前 5 回合，发现退化立即 send_message 矫正。
