# 从交互到剧场：改写手艺（compile-from-interactions）

> 这是 Director 的"改稿手艺"能力册。
> 回答的是："Team 演完了，我手上有 interactions.jsonl（N 个 Agent 各自的第一人称回合），我怎么把它改写成一份第三人称戏剧白描 novel.md？"
>
> 这一步是 Phase 3 编译的核心。不是拼接，是**熔合**。

---

## 核心原则：交互是素材，novel 是成品

### 素材 vs 成品的差距

**interactions.jsonl 里每条回合长这样**：

```json
{
  "agent": "lin-mo",
  "turn": 3,
  "content": "他坐下。两只手放在膝盖上。'跟前天一样。前天跟大前天一样。这就是精神病院的好处。'他自己说完停半秒。他没笑。",
  "pov": "first-person"  // 林墨视角，用"他"指代自己是因为 Agent 输出时已按回合制规范做了第三人称化（见 interaction-craft.md）
}
```

```json
{
  "agent": "li-doctor",
  "turn": 4,
  "content": "他看着林墨。他把左手从病历上挪开。他合上病历。'林医生。'他停了。'林墨。'"
}
```

**novel.md 最终成品长这样**：

```
他坐下。两只手放在膝盖上。

"跟前天一样。"他说，"前天跟大前天一样。这就是精神病院的好处。"

他自己说完停了半秒。他没笑。

李医生没接话。他看着他。他把左手从病历上挪开。他合上病历。

"林医生。"他停了。

"林墨。"
```

**差距**：
- 素材里每条回合是独立的（按 agent 分）
- 成品里是**一个叙述者**在看（按时间顺序）
- 成品里**对白被提取出来**成一个独立段落格式
- 成品里加了**微量的环境呼吸**（空行节拍）
- 成品里**没有 inner_thought 技术标签**

---

## 改写三原则

### 原则 1 · 按时间熔合，不按角色分段

**错误做法**（独白拼图）：
```
林墨
（他所有回合的内容拼起来）

李医生
（他所有回合的内容拼起来）
```

**正确做法**（时间熔合）：
```
（回合 0-1 的内容）
（回合 2-3 的内容）
（回合 4-5 的内容）
……
```

按 turn 升序遍历 interactions.jsonl，按发生时间把每个 Agent 的回合编织在一起——就像剧场观众看到的一样，场上同时有 N 个人，镜头按时间移动。

### 原则 2 · 第一人称回合 → 第三人称叙事

Agent 输出时按协议已经用"他/她"指代自己（这是回合制协议的约定）。你**不用做人称转换**，只需要：

- 把每个 Agent 的回合**打平到同一叙述层**
- 删掉 Agent 输出里冗余的"我"/"你"（它们是 Agent 思考时的习惯残留）
- 合并连续同人称句子为一段

**示例**：

Agent 素材：
```
他看着林墨。他把左手从病历上挪开。他合上病历。
```

成品：
```
李医生看着他。他把左手从病历上挪开。合上病历。
```

注意：
- 把"林墨"改为"他"（叙述里林墨是主视角）
- "他把……他合……"合并为"他把……合……"（去掉重复主语）

### 原则 3 · 对白是"硬骨头"，不改只排

对白引号内的话是**Agent 的产物，不是 Director 的产物**。改写时：
- ✅ 保留对白原文（最多做语气词微调）
- ✅ 改写对白前后的动作描写
- ✅ 重新切分对白的段落布局
- ❌ 修改对白内容
- ❌ 把对白改为间接引语（除非删除它）

## 四个改写技法

### 技法 1 · 对白的"三段格式"

一段对白的标准排法：

```
（前置动作或身体反应）

"对白内容。"（可选的说话者标记）

（后置反应或沉默）
```

**示例**：

素材：
```
李医生 turn 4: 他合上病历。'林医生。'他停了。'林墨。'
林墨 turn 5: 他抬眼。
```

成品：
```
李医生合上病历。

"林医生。"

他停了。

"林墨。"

他抬眼。
```

注意三段格式：
- 合病历（前置动作）
- "林医生。"（对白 1）
- 他停了（中间呼吸）
- "林墨。"（对白 2）
- 他抬眼（反应）

### 技法 2 · 内心闪念的融入

Agent 输出的 `[inner_thought]` 标签**不能出现在 novel.md 里**。怎么处理？

**方案 A · 自由间接引语**（最推荐）：

素材：
```
[inner_thought] 这句很新。三年我没听过他这么锋利的话。
```

成品：
```
这句话很新。三年他没听过这种话。
```

注意：
- 去掉 [inner_thought] 标签
- 用"他"代替"我"
- 嵌入到叙事段落里（不独立成段）
- 保留内心的"口吻"（"这句话很新"这种感慨性语气就是视角的暗示）

**方案 B · 身体动作外化**（有时比 A 更好）：

素材：
```
他推门出去。
[inner_thought] 对不起。不对，不是对不起。没关系。没关系。
```

成品：
```
他推门出去。门关得很轻。他在门外的走廊上站了两秒才往前走。
```

注意：**把内心动摇外化为身体动作**——"站了两秒才走"就是"对不起"的身体表达。

**方案 C · 删除**（如果内心闪念只是复述了已经发生的事实，删掉）：

素材：
```
他说"好"。
[inner_thought] 我听见了。我说了好。
```

成品：
```
他说"好"。
```

### 技法 3 · 多人同场的视角调度

群戏里多个 Agent 同时有动作，怎么在第三人称叙事里保持清晰？

**铁律：每 200 字最多换 1 次视角锚点**。

一段里不断切换视角（从林墨看周伯，切到周伯看林墨，切到张教授看两人）= 读者晕。

**合法做法**：

```
林墨看见红塔山在桌上。他把目光收回来。
（主视角：林墨）

周伯看见他的目光停住了又收回。他没说这盒烟是怎么回事。
（切到周伯，但只切一拍——通过"看见他"这个动作）

林墨捧着茶。茶到嗓子里他停了一下再咽。
（切回林墨，200 字内不再切）
```

每次视角切换必须有一个**物理锚点**（"他看见""他听见""他发现"），不能凭空跳。

### 技法 4 · 环境呼吸

world-manager 的 broadcast 是**环境素材**。不要整段原样照搬——**拆散分散到场景里**做呼吸节拍。

素材：
```
world-manager broadcast: 面谈室在二层东南角，是全院日照最早的一间。八点二十分这时候太阳还没越过对面那栋楼的屋脊，室内是漫反射的灰白光。桌上那份复核病历摆得比平常偏右一寸半。
```

❌ 错误用法（整段开头塞进去）：
```
面谈室在二层东南角，是全院日照最早的一间。八点二十分这时候太阳还没越过对面那栋楼的屋脊，室内是漫反射的灰白光。桌上那份复核病历摆得比平常偏右一寸半。

李医生 08:15 就坐下了。
```

✅ 正确用法（拆散呼吸）：
```
八点二十。面谈室。

李医生 08:15 就坐下了。窗没开。桌上的复核病历摆得比平常偏右一寸半。

他左手按在右上角。

对面那栋楼的屋脊还没被太阳翻过。室内是漫反射的灰白光。
```

环境描写像吐气——在动作之间短促插入，不是段落级别堆砌。

## 改写工作流

### 步骤 1 · 读取 interactions.jsonl

按 turn 升序遍历所有回合。

### 步骤 2 · 切分场景

按 world-manager 的 "broadcast 开场" 和 "broadcast 收场" 切分。每个场景成为 novel.md 的一个"章节段"（用 `一 / 二 / 三` 中文数字标识，不用 `##` 标题）。

### 步骤 3 · 场景内部熔合

对每个场景内部：

1. **提取主视角**（beat-sheet 已标）
2. **按 turn 顺序处理每个回合**：
   - Agent 回合 → 改写为第三人称叙事 + 对白段
   - world-manager broadcast → 拆散成呼吸
3. **删除 [inner_thought] 标签**，按技法 2 处理（融入 / 外化 / 删除）
4. **合并重复主语**的连续句子
5. **调整段落呼吸**（对白段短、叙事段 3-6 句、爆点段偶尔独占）

### 步骤 4 · 场景间过渡

场景和场景之间用一个空行分隔。如果是大跨度切换（几个小时后 / 不同地点）用**中文数字编号**：

```
……他推门出去。门关得很轻。

二

九点四十。会客室。

……
```

### 步骤 5 · 通篇检查

交付前读一遍，检查：
- 无 `[inner_thought]` `[action]` 等技术标签
- 无 `林墨\n\n` `李医生\n\n` 角色名小标题
- 叙述段落以"他/她"开头（不是"我"）
- 对白引号内外衔接顺滑
- 环境描写是呼吸而不是开场长段

然后跑 `check-ai-taste.js` 自动门控。

## 常见改写错误对照

### 错误 1 · 把内心想法当叙述

素材（Agent 输出）：
```
他咽回去了。我本来想说的是柜子。
```

❌ 错误成品：
```
他咽回去了。我本来想说的是柜子。
```

✅ 正确成品：
```
他咽回去了。柜子那个字没出口。
```

或用自由间接引语：
```
他咽回去了。他本来想说的是柜子。
```

### 错误 2 · 人名开头独立成段的独白小标题

素材：
```
agent: "lin-mo", turn: N, content: "……"
agent: "li-doctor", turn: N+1, content: "……"
```

❌ 错误成品：
```
林墨

他坐下。

李医生

李医生看着他。
```

✅ 正确成品：
```
林墨坐下。

李医生看着他。
```

（人名作为句子的主语，不作为标题）

### 错误 3 · 保留 Agent 的"我"自述

素材：
```
[agent: lin-mo] 我把饭盒拎到前台。"麻烦转交，"我说。"写她的名字。"
```

❌ 错误成品：
```
我把饭盒拎到前台。"麻烦转交，"我说。"写她的名字。"
```

✅ 正确成品：
```
他把饭盒拎到前台。"麻烦转交。"他说。"写她的名字。"
```

注意：对白引号内部不用改（如果引号里有"我"比如"我今天要跟周伯伯出一趟门"那是合法的——角色在说自己）。

## 给 Director 的自检清单

改写完成后问自己：

- [ ] 我是按 turn 时间序改写的，不是按角色分区改写的？
- [ ] 叙述段落主语全部是"他/她"？
- [ ] 所有 `[inner_thought]` 标签都处理掉了（融入 / 外化 / 删除）？
- [ ] 对白内容没改（只改了排版 + 前后动作）？
- [ ] 环境描写被拆散到场景里作呼吸，而不是开头长段铺垫？
- [ ] 每 200 字最多切一次视角？
- [ ] 场景之间用中文数字编号，不用 `##` 标题？

七问全过 → 送 `check-ai-taste.js` 门控 → 进 Phase 4 Critic。

---

## 手艺的哲学

> interactions.jsonl 是**演员在镜头前做了什么**；
> novel.md 是**剪辑师看完素材后决定观众看到什么**。
>
> Director 在 Phase 2 是摄影棚里的导演。
> Director 在 Phase 3 是剪辑室里的剪辑师。
> 两种工作的肌肉不同。不要混淆。
